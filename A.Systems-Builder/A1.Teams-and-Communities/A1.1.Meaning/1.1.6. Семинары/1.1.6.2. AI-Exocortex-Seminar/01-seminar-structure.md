# Семинар: Работа с ИИ-экзокортексом

> РП #12 | Формат: 90 мин (онлайн/офлайн) | Аудитория: клуб ЛР
> Принцип: от боли → к простому → к сложному → к «хочу так же»

---

## Логика погружения

Аудитория **не знает** терминов Pack, Downstream, FPF. Они знают ChatGPT и Notion.
Стратегия: **5 ступеней**, каждая отвечает на вопрос предыдущей.

```
Ступень 1: "У меня так же"        → Боль (все теряют контекст)
Ступень 2: "А можно по-другому?"   → Идея экзокортекса (что это)
Ступень 3: "Как это выглядит?"     → Живое демо (рабочая сессия)
Ступень 4: "Как это устроено?"     → Архитектура (слои, агенты, Pack)
Ступень 5: "Как мне начать?"       → Минимальный экзокортекс за 30 мин
```

## СТУПЕНЬ 1. Боль (10 мин)

### Слайд 1: Титульный

```
в продолжение своего мышления

```

### Слайд 2: Знакомая ситуация

```
Понедельник:
  "Claude, помоги спланировать неделю"
  → Отличный план!

Среда:
  "Claude, как продвигается план?"
  → "Какой план? Я вас не помню."

Пятница:
  "Claude, напиши отчёт за неделю"
  → Галлюцинации о том, чего не было
```

**Спикер говорит:** «Знакомо? ИИ — мощный, но у него память золотой рыбки. Каждый разговор — с нуля. Вы тратите 20% времени на ввод контекста, который уже объясняли.»

### Слайд 3: Три проблемы ИИ-ассистента

```
1. АМНЕЗИЯ        Новая сессия = чистый лист
2. ГАЛЛЮЦИНАЦИИ   Нет проверенных знаний — додумывает
3. ОДИН НА ВСЁ    Один ассистент и планирует, и кодит,
                   и пишет — и ничего не делает хорошо
```

### Пример для демо (Ступень 1)

## СТУПЕНЬ 3. Живое демо: рабочая сессия (25 мин)

> **Главный блок семинара.** Показываем реальную работу, не слайды.

### Слайд 7: Что сейчас увидите

```
ЖИВАЯ ДЕМОНСТРАЦИЯ: одна рабочая сессия

1. Открытие  — ИИ проверяет задачу по плану (WP Gate)
2. Работа    — ИИ захватывает знание (Capture)
3. Закрытие  — ИИ формализует и фиксирует результат

Время: ~15 минут реальной работы
```

### Демо-сценарий (подготовить заранее)

**Подготовка:** Создать тестовую задачу в MEMORY.md (например, РП #99 «Демо для семинара»).

#### Акт 1: Открытие сессии (5 мин)

**Что делаем:** Открываем Claude Code и говорим:
> «Добавь в бота приветственное сообщение на казахском языке»

**Что аудитория видит:**
1. Claude читает MEMORY.md → находит (или не находит) РП
2. Claude говорит: *«Этой задачи нет в плане. Вот ваши текущие РП: [таблица]. Записать новый?»*
3. Мы говорим: «Да, РП #99, бюджет 15 мин»
4. Claude записывает в MEMORY.md + WeekPlan + inbox
5. Claude говорит: *«Работа: добавить kz-приветствие. Рабочий продукт: i18n-файл. Метод: редактирование.»*

**Что объясняем:** «ИИ сам проверил, что задача в плане. Не я вспомнил — он вспомнил за меня. Это WP Gate — блокирующее правило. Без него я бы начал работать над чем попало.»

#### Акт 2: Работа + Capture (5 мин)

**Что делаем:** Claude начинает работу, читает файлы бота, находит i18n-систему.

**Что аудитория видит:**
1. Claude читает файл, делает изменение
2. В процессе обнаруживает паттерн и анонсирует:
   *«Capture: i18n fallback chain (kz → ru → en) → PACK-digital-platform (method)»*
3. Мы одобряем или откладываем

**Что объясняем:** «Заметили? ИИ не просто сделал задачу — он обнаружил знание. Паттерн, который можно использовать в другом проекте. И сразу предложил записать. Это Capture-to-Pack — во время работы, не после.»

#### Акт 3: Закрытие (5 мин)

**Что делаем:** Говорим «закрываю сессию».

**Что аудитория видит:**
1. Claude собирает captures
2. Формирует отчёт:
   ```
   РП: #99 — Демо для семинара
   Статус: done
   Сделано: добавлен kz в i18n
   Captures: 1 → PACK-digital-platform (method)
   Git: закоммичено + запушено ✅
   ```
3. Обновляет MEMORY.md, Plan, делает backup

**Что объясняем:** «Сессия закрыта. Знание зафиксировано. План обновлён. Backup сделан. Всё автоматически. Завтра я открою новую сессию — и ИИ будет знать, что было сегодня.»

### Слайд 10: 4 ИИ-агента

```
СТРАТЕГ (Strategist)
  Что: Планирует неделю, день, ревьюит заметки
  Когда: Автоматически (пн 4:00, каждый день)
  Результат: WeekPlan, DayPlan, WeekReport
  6 операционных навыков

ЭКСТРАКТОР (Knowledge Extractor)
  Что: Превращает информацию в формализованные знания
  Когда: Закрытие сессии, каждые 3 часа (inbox)
  Результат: Pack-сущности с ID и связями
  7 процессов извлечения

СИНХРОНИЗАТОР (Synchronizer)
  Что: Наблюдает, собирает, маршрутизирует, уведомляет
  Когда: Каждые 2 мин (файлы), ежедневно (код)
  Результат: Проекции Pack → Downstream
  4 функции: file-sync, code-scan, pack-project, notify

АГЕНТ РАЗВЁРТЫВАНИЯ (Setup Agent)
  Что: Создаёт экзокортекс для нового пользователя
  Когда: По запросу (fork + configure)
  Результат: Рабочий экзокортекс за 1 сессию
```

## СТУПЕНЬ 5. Как начать (15 мин)

### Слайд 15: Минимальный экзокортекс (3 файла)

```
~/MyProject/
├── CLAUDE.md      ← Правила для ИИ (10 строк)
├── MEMORY.md      ← Что помнить между сессиями (5 строк)
└── memory/
    └── goals.md   ← Ваши цели (3 строки)
```

### Пример для демо (Ступень 5)

> **Живой показ (5 мин):** Создать минимальный экзокортекс прямо на экране.

**Файл 1: CLAUDE.md** (показать на экране)

```markdown
# Правила

## Работа
- Перед началом проверь MEMORY.md — что мы делали в прошлый раз?
- После работы обнови MEMORY.md — что сделали сегодня?

## Мои проекты
- Диссертация: ~/dissertation/
- Курс по ML: ~/ml-course/
```

**Файл 2: MEMORY.md** (показать на экране)

```markdown
# Память

## Текущие задачи
| # | Задача | Статус |
|---|--------|--------|
| 1 | Глава 3 диссертации | в работе |
| 2 | Лекция по нейросетям | черновик готов |

## Прошлая сессия
Написали outline главы 3. Осталось: секция 3.2 (эксперименты).
```

**Файл 3: memory/goals.md** (показать на экране)

```markdown
# Цели на февраль
- Сдать главу 3 научруку до 20 февраля
- Подготовить лекцию до 15 февраля
```

**Что объясняем:** «Всё. 3 файла, 20 строк. Откройте Claude Code в этой папке — и ИИ уже знает ваши задачи, помнит прошлую сессию, проверяет план. Это минимальный экзокортекс. Мой вырос из такого же.»
