# Executive Summary: AI Exocortex — IDE для интеллекта

## 1. Problem

**LLM-тьюторы галлюцинируют.** ChatGPT отвечает на любой вопрос, но не отвечает за качество ответа. Нет верифицированной базы знаний — есть статистическая модель, которая звучит убедительно.

**Курсы не адаптируются.** Один контент для всех. Нет связи между тем, что человек уже знает, и тем, что ему нужно дальше. Completion rate онлайн-курсов — 5-15%.

**Заметки мертвы без привычки.** PKM-системы (Obsidian, Notion, Roam) требуют дисциплины, которой нет у 95% людей. Знания копятся, но не работают.

**Итог:** Человек потребляет контент, но не развивается. Измеряет «прочитал 3 главы», а не «что делаю иначе».

---

## 2. Solution

**AI Exocortex** — IDE для интеллекта. Среда, в которой формализованные знания подаются персонально через цифрового двойника.

Три ключевых отличия:

1. **Verified Knowledge Base (Pack)** — знания формализованы по стандарту SPF: сущности с типами, связями, версиями. Не промпт, а структурированная предметная область. Галлюцинации исключены — бот работает только с верифицированными знаниями.

2. **Digital Twin** — цифровой двойник ученика: цели, текущий уровень, прогресс, контекст. Система знает, что человек уже знает, и подбирает следующий шаг.

3. **Template-Exocortex** — любой эксперт разворачивает свою мастерскую на нашей инфраструктуре за 1 день: Pack (знания) → Bot (подача) → Community (среда).

## 3. Product

**Точка входа:** Telegram-бот (@aist_me_bot).

**Три режима:**
- **Марафон** — 14-дневный интенсив по системному мышлению
- **Лента** — ежедневная подборка по выбранным темам (адаптивная)
- **Консультант** — диалог с AI по верифицированной базе знаний (tool_use: knowledge-mcp + digital-twin-mcp + guides-mcp)

**Инфраструктура:**
- 5300+ документов проиндексировано (9 источников: 4 Pack + 5 downstream)
- Hybrid search: pgvector (semantic) + pg_trgm (keyword) + tsvector (FTS)
- MCP-серверы на Cloudflare Workers (knowledge, digital-twin, guides)
- Neon Serverless Postgres, Railway (EU Amsterdam)

**Масштабирование:**
- Template-exocortex: GitHub template repo → fork → setup.sh → работающий экзокортекс с ботом
- 5 тиров обслуживания (T1 self-service → T5 enterprise)

## 5. Moat

1. **Pack-система** — structured domain knowledge по стандарту SPF. Не копируется промптом: каждая сущность имеет тип, код, связи, версию, аудиторию. Создание Pack требует экспертизы + формализации.

2. **SPF Framework** — стандарт организации знаний (кодирование сущностей, масштабируемость Pack, валидация). Open-source, но экосистема строится вокруг.

3. **Методология МИМ** — Мастерская Интеллектуального Мастерства. Системное мышление + практика развития. Контент, который нельзя сгенерировать — он создаётся экспертами.

4. **Digital Twin** — с каждым взаимодействием система знает человека лучше. Переключение на конкурента = потеря персональной истории.

## 6. Business Model

**B2C: Подписка через Telegram Stars**
- Бесплатный тир: марафон + ограниченная лента
- Платный тир: полная лента + консультант + персонализация
- Unit economics: Neon + CF Workers = ~$0.01/user/month hosting

**B2B: Platform (Template-Exocortex)**
- Эксперт создаёт Pack → разворачивает экзокортекс → получает бота с персонализацией
- Модель: комиссия с подписок учеников эксперта
- 5 тиров: от self-service (бесплатно) до enterprise (кастом)

**Рычаг:** Каждый новый эксперт приносит свою аудиторию + свои знания → сетевой эффект.
