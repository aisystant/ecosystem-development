# Release 2: Объём и критерии готовности

## Цель релиза

Привязка к реальному пользователю, персональные данные, недельный планировщик и итоги.

---

## Что входит в релиз

### 1. Идентификация
- Сервис идентификации (OAuth 2.1)
- Выдача и обновление токенов
- Реестр привязок "пользователь ↔ twin_id"
- Проверка токена в MCP-сервере

### 2. UI
- Экран подключения / логина
- Обработка ошибок авторизации (токен истёк, нет доступа)
- Состояние "не подключён"

### 3. Недельный планировщик
- MCP-инструмент `build_week_plan(помидорки, дни)`
- MCP-инструмент `save_week_plan`
- UI: экран планирования (ввод 2 чисел, показ вариантов)

### 4. Недельные итоги
- MCP-инструмент `save_week_review`
- UI: экран "Итоги недели"

---

## Критерии готовности

| # | Критерий | Проверка |
|---|----------|----------|
| 1 | Пользователь может залогиниться | OAuth flow работает |
| 2 | После логина видны персональные данные | Данные соответствуют twin_id |
| 3 | Можно создать план недели | Инструмент возвращает варианты |
| 4 | Можно сохранить план | План записывается в ЦД |
| 5 | Можно сохранить итог недели | Итог записывается в ЦД |
| 6 | Токен истёк → показывается экран реавторизации | UI обрабатывает 401 |
