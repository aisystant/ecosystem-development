# Концепция ИИ-платформы

## 0. Назначение документа

Документ описывает **концепцию ИИ-платформы** — операционной системы для развития человека в экосистеме. Определяет назначение, архитектурные принципы и ключевые компоненты платформы.

**Адресаты:**
- Архитекторы и разработчики экосистемы
- Руководители продуктов и направлений
- Инвесторы и стратеги
- Методологи и кураторы

---

## 1. Видение и миссия ИИ-платформы

### 1.1. Что такое ИИ-платформа

**ИИ-Платформа** — это **операционная система** для развития человека, которая:
- Интегрирует все ИТ-системы экосистемы (LMS, Клуб, CRM, Биллинг)
- Обеспечивает персональное сопровождение каждого участника
- Собирает, анализирует и использует данные о траектории развития
- Координирует работу ИИ-ассистентов и агентов
- Начисляет токены за активности и стимулирует развитие

### 1.2. Место платформы в экосистеме

ИИ-платформа является одновременно:
- **Инструментом для экосистемы** — обеспечивает функционирование всех подсистем
- **Частью экосистемы** — развивается вместе с участниками и процессами

### 1.3. Ключевые принципы

1. **Человек в центре** — вся платформа работает для развития конкретного человека
2. **Единое состояние** — Цифровой двойник как источник правды о траектории
3. **Персонализация** — каждый участник получает уникальный опыт
4. **Прозрачность** — человек понимает, как платформа работает и почему предлагает те или иные действия
5. **Стимулы через токены** — активность вознаграждается, пассивность не поощряется

## 2. Архитектура платформы

### 2.1. Слоистая архитектура

ИИ-платформа построена по **слоистой архитектуре**:

```
┌─────────────────────────────────────────────────────────┐
│  СЛОЙ 5: ИНТЕРФЕЙСЫ                                     │
│  ChatGPT Apps │ Telegram │ Web LMS │ Mobile │ API      │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  СЛОЙ 4: ИИ-АССИСТЕНТЫ (человеко-ориентированные)      │
│  Проводник │ Ассистент Интеллектуала │ ...              │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  СЛОЙ 3: ИИ-АГЕНТЫ (системно-ориентированные)          │
│  17 агентов: Orchestrator │ Ingestor │ Planner │ ...    │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  СЛОЙ 2: ИИ-СИСТЕМЫ (инфраструктурные подсистемы)      │
│  Twin │ Hub │ Токены │ Граф │ Аналитика │ ...          │
└─────────────────────────────────────────────────────────┘
                          ▼
┌─────────────────────────────────────────────────────────┐
│  СЛОЙ 1: ПЛАТФОРМЕННАЯ ИНФРАСТРУКТУРА                   │
│  MCP │ Event Bus │ Auth │ Storage │ Observability      │
└─────────────────────────────────────────────────────────┘
```

### 2.2. Ключевые компоненты по слоям

#### Слой 1: Инфраструктура и детерминированные ИТ-системы

**Платформенная инфраструктура:**
- **MCP-инфраструктура** — Model Context Protocol для связи компонентов
- **Event Bus** — шина событий для асинхронной коммуникации
- **Observability** — мониторинг, логи, трассировка

**Детерминированные ИТ-системы (ai_role: none/feature):**
- **Цифровой двойник (Twin)** — хранение состояния, траектории, артефактов (SurrealDB)
- **Хаб активностей** — сбор и агрегация событий (Kafka/Flink)
- **Распределитель токенов** — начисление токенов по правилам (Rules Engine)
- **Централизованное хранилище** — общее хранилище данных (SurrealDB)
- **Идентификация и доступ (ORY)** — аутентификация и авторизация
- **CRM, LMS, Клуб, Биллинг** — прикладные системы экосистемы

#### Слой 2: ИИ-Системы (ai_role: core)
- **Эпистемический граф** — структура знаний, рассуждения через ML
- **Система аналитики** — метрики, инсайты через ML-модели
- **Система персонализации** — адаптация контента через ML

#### Слой 3: ИИ-Агенты
17 агентов, организованных в 5 групп:
- Ядро выполнения и координации (3 агента)
- Сбор фактов (3 агента)
- Обучение и развитие (6 агентов)
- Поддержка ритма (2 агента)
- Аналитика и безопасность (3 агента)

Подробно описаны в [[Карта ИТ-систем 3.2]]

#### Слой 4: ИИ-Ассистенты
- **Проводник (Ассистент Ученика)** — персональный маршрут развития
- **Ассистент Интеллектуала** — компиляция знаний (планируется)
- **Ассистент Профессионала** — применение знаний (планируется)
- **Ассистент Исследователя** — научный метод (планируется)
- **Ассистент Просветителя** — передача знаний (планируется)

#### Слой 5: Интерфейсы
- **ChatGPT Apps (Apps SDK)** — приложение внутри ChatGPT
- **Telegram-бот** — доступ через мессенджер
- **Web LMS** — встроенный чат в платформе обучения
- **Клуб** — встроенный чат в сообществе
- **Mobile App** — мобильное приложение (планируется)
- **API** — программный доступ для интеграций

## 3. Две ключевые фишки платформы

### 3.1. Фишка 1: Механизм масштабирования через токены

**Суть:** участники получают токены за активные действия по развитию себя и экосистемы.

**Как работает:**
1. **Источники активности:** обучение, проекты, публикации, продажи, peer reviews, коммуникации
2. **Сбор событий:** Хаб активностей собирает все события от всех подсистем
3. **Начисление токенов:** Распределитель токенов применяет правила и начисляет токены
4. **Кошельки:** токены попадают на личные кошельки участников
5. **Использование:** токены можно тратить на курсы, консультации, премиум-функции

**Эффект:**
- Стимулирует активность и созидание
- Выравнивает возможности (токены можно заработать, а не только купить)
- Создает экономику внутри экосистемы
- Масштабирует через саморегуляцию

### 3.2. Фишка 2: Персональная траектория с цифровым двойником

**Суть:** у каждого участника есть Цифровой двойник, который хранит всю траекторию развития.

**Что хранит Цифровой двойник:**
- **События (E.*)**: все действия участника (обучение, задания, активность в клубе)
- **Слоты (SLT.*)**: слоты времени, выделенные на саморазвитие
- **Рабочие продукты (W.*)**: артефакты (конспекты, модели, решения)
- **Маршруты (ROUTE.*)**: планы и шаги развития
- **Ступени (STAGE.*)**: текущая ступень (Случайный, Практикующий, Систематический и т.д.)
- **Метрики (MET.*)**: агрегированные показатели (часы в неделю, регулярность)

**Как используется:**
1. **Проводник** читает Twin и формирует персональные рекомендации
2. **Аналитика** строит инсайты о прогрессе
3. **Методист** видит паттерны и улучшает методику
4. **Оценщик** проверяет артефакты и обновляет ступень

**Эффект:**
- Полная персонализация опыта
- Непрерывность траектории (даже если участник ушел и вернулся)
- Прозрачность прогресса для человека
- Основа для аналитики и улучшения методики

## 4. Ключевые сценарии использования платформы

### 4.1. Сценарий 1: Старт развития (онбординг)

```
Человек регистрируется → создается Цифровой двойник →
Проводник проводит первичную оценку →
формируется начальный маршрут →
человек получает первые задания
```

### 4.2. Сценарий 2: Ежедневная работа с Проводником

```
Человек пишет Проводнику "Что делать сегодня?" →
Проводник читает Twin (слоты, задания, активность) →
Проводник читает методику (ступени, практики) →
Проводник формирует план дня →
человек выполняет задания →
события записываются в Twin →
начисляются токены
```

### 4.3. Сценарий 3: Переход между ступенями

```
Человек стабильно удерживает ритм 4+ недели →
Аналитик выявляет паттерн →
Проводник инициирует переоценку ступени →
Проводник предлагает новые практики →
маршрут обновляется под новую ступень
```

### 4.4. Сценарий 4: Сбой ритма и возврат

```
Человек пропускает слоты несколько дней →
RhythmKeeper фиксирует сбой →
Проводник пишет "Заметил, что активность снизилась. Что случилось?" →
человек объясняет причину →
Проводник адаптирует план под текущую ситуацию →
постепенное возвращение к ритму
```

### 4.5. Сценарий 5: Вклад в экосистему

```
Человек публикует статью в клубе →
Хаб активностей фиксирует событие →
Распределитель токенов начисляет токены →
Аналитик обновляет метрики вовлеченности →
Twin обновляется (событие + токены)
```

## 5. Технологический стек платформы

### 5.1. Основные технологии

| Компонент | Технология | Назначение |
|-----------|-----------|------------|
| **Цифровой двойник** | SurrealDB | Graph + Document DB для траекторий |
| **MCP-серверы** | Python/TypeScript | Интерфейсы к данным и методике |
| **Методика** | Git-репозиторий (Markdown) | Версионируемая база знаний |
| **Event Bus** | Kafka / RabbitMQ | Шина событий |
| **ИИ-модели** | GPT-4o, Claude 3.5 | LLM для ассистентов и агентов |
| **Идентификация** | ORY Kratos/Hydra | SSO, OAuth2 |
| **API Gateway** | Kong / Envoy | Единая точка входа |
| **Observability** | OpenTelemetry + Grafana | Мониторинг и трассировка |

### 5.2. Интеграции с существующими системами

- **LMS (Moodle/Canvas)** — через Events API
- **Клуб (Discourse/Circle)** — через Webhooks
- **CRM** — через REST API
- **Биллинг** — через Webhooks
- **Календарь** — через CalDAV
- **Git** — прямое чтение репозитория методики

## 6. Принципы разработки и эволюции платформы

### 6.1. Инкрементальное развитие

Платформа развивается **итеративно**:
1. **MVP**: Проводник + Цифровой двойник + базовые агенты
2. **V2**: Полный набор агентов + токены
3. **V3**: Ассистенты для других ролей
4. **V4**: Расширенная аналитика и рекомендации

### 6.2. Открытость и расширяемость

- **Новые агенты** можно добавлять без изменения ядра
- **MCP** позволяет интегрировать новые источники данных
- **Event Bus** позволяет подключать новые системы
- **API** открыт для сторонних разработчиков

### 6.3. Observability и улучшение

- Все действия агентов логируются
- Метрики собираются в реальном времени
- A/B-тестирование рекомендаций Проводника
- Обратная связь от людей анализируется и улучшает методику

## 7. Метрики успеха платформы

### 7.1. Метрики на уровне человека
- **Регулярность**: % дней со слотами саморазвития
- **Объем**: средние часы в неделю на обучение
- **Прогресс**: переходы между ступенями
- **Артефакты**: количество и качество рабочих продуктов

### 7.2. Метрики на уровне экосистемы
- **Активных участников**: количество людей с регулярной активностью
- **Токенов начислено**: общий объем начислений
- **Транзакций токенов**: активность экономики
- **Вовлеченность**: активность в клубе, LMS, проектах

### 7.3. Метрики на уровне платформы
- **Uptime**: доступность систем (99.9%+)
- **Latency**: время отклика ассистентов (<2s)
- **Точность рекомендаций**: % полезных рекомендаций Проводника
- **Качество данных**: % корректных записей в Twin
