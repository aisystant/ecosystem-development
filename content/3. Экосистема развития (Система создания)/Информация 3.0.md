# 4.0. Информация о разделе "Системы"

Этот документ поясняет назначение раздела «Системы»: здесь описаны подсистемы экосистемы, их цели, границы, данные, интерфейсы и связи.

## О чём эта папка

**Назначение:** Архитектурная документация всех подсистем экосистемы. Каждая подсистема описана как:
- Назначение и границы ответственности
- Ключевые компоненты и процессы
- Интеграции с другими подсистемами
- Основные артефакты и данные
- Технологический стек
- Roadmap развития

**Связь с другими разделами:**
- [[0. Управление]] — решения по архитектуре (ADR), дорожная карта подсистем
- [[1. Идеи развития экосистемы]] — концепция функционирования, принципы развития
- [[3. Библиотека]] — методология и знания, используемые в подсистемах

Подробную карту подсистем с назначением и интеграциями смотрите в [[4.1. Список подсистем]].

## Как пользоваться

### Добавление новой подсистемы

1. Создайте файл с именем `4.N. Название подсистемы.md` в папке `content/4. Системы/`
2. Добавьте frontmatter с обязательными полями:
   ```yaml
   ---
   type: doc
   audience: manual
   edit_mode: manual
   layer: architecture
   scope: global-core
   security: internal
   status: draft|in_progress|active|archived
   version: 1.0
   created: YYYY-MM-DD
   ---
   ```
3. Используйте следующую структуру документа:
   - Назначение
   - Ключевые компоненты
   - Процессы
   - Интеграции
   - Основные артефакты/данные
   - Технологический стек
   - API и контракты
   - Roadmap

### Обновление существующей подсистемы

1. Внесите изменения в соответствующий файл
2. Обновите поле `version` в frontmatter
3. Добавьте запись в раздел "История изменений" (если есть)
4. Обновите связанные документы при необходимости
