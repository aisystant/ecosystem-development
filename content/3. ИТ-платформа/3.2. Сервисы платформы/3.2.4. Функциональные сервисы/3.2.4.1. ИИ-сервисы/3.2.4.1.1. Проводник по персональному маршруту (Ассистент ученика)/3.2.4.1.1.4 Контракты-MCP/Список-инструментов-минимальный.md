# Список MCP-инструментов (минимальный)

Инструменты для Release 1 (демо без идентификации).

---

## 1. get_week_state

**Назначение:** Получить состояние недели — активные действия, дни с активностью.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| demo_profile | string | Нет | `empty`, `weak`, `good` (для демо) |

**Ответ:** см. `Контракт_состояние-недели.json`

---

## 2. get_week_time

**Назначение:** Получить инвестированное время за неделю и среднее за день.

**Параметры:**
| Параметр | Тип | Обязательный | Описание |

## Инструменты для Release 2

| Инструмент | Назначение |
|------------|------------|
| `build_week_plan` | Собрать план недели по двум числам (помидорки/день, дней/неделя) |
| `save_week_plan` | Сохранить выбранный план недели |
| `save_week_review` | Сохранить итог недели |
| `configure_slot` | Настроить слот саморазвития (время, дни, длительность) |
| `save_product` | Сохранить рабочий продукт (заметка, черновик, заготовка) |

---

## Принципы контрактов

1. **Структурированные ответы** — JSON с предсказуемой схемой
2. **Демо-режим** — все инструменты поддерживают параметр `demo_profile`
3. **Без идентификации в R1** — `twin_id` фиксирован или определяется по `demo_profile`
4. **Версионирование** — версия контракта указана в каждом JSON-файле
