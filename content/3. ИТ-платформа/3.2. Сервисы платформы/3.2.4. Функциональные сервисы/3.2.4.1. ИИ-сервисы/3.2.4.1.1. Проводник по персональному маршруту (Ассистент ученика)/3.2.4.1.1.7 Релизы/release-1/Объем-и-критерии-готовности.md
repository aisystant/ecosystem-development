# Release 1: Объём и критерии готовности

## Цель релиза

Работающий прототип в ChatGPT, который показывает недельные данные демо-пользователя и даёт рекомендации.

## Ключевое ограничение

**Без идентификации.** MCP-сервер работает с фиксированным демо-двойником (`demo_twin_id`).

---

## Что входит в релиз

### 1. UI на Apps SDK
- Экран "Моя неделя"
- Экран "Слот саморазвития"
- Экран "Рабочие продукты"
- Карточка рекомендации ассистента
- Обработка состояний: загрузка, ошибка, пустые данные

### 2. MCP-сервер
- `get_week_state` — состояние недели (действия, дни с активностью)
- `get_week_time` — инвестированное время за неделю
- `get_slot_consistency` — систематичность слота саморазвития
- `get_week_products` — рабочие продукты за неделю
- Демо-режим: возврат данных без идентификации

### 3. Цифровой двойник
- Демо-данные для сценариев:
  - Пустая неделя (нет активности)
  - Слабая неделя (1-2 дня, мало времени)
  - Хорошая неделя (4+ дня, стабильный слот)

### 4. Методология ассистента
- Правила интерпретации недельных метрик
- Шаблон ответа "один экран"
- Формулировки рекомендаций для каждого сценария

---

## Что НЕ входит в релиз

- Идентификация пользователя (OAuth, токены)
- Персональные данные (только демо)
- Запись данных (только чтение)
- Планировщик недели
- Итоги недели
- Токен-экономика

## Критерии готовности

### Функциональные

| # | Критерий | Проверка |
|---|----------|----------|
| 1 | Пользователь открывает приложение в ChatGPT | UI загружается без ошибок |
| 2 | Отображается экран "Моя неделя" | Видны метрики демо-двойника |
| 3 | MCP-инструменты возвращают данные | Все 4 инструмента работают |
| 4 | Ассистент даёт рекомендацию | Рекомендация соответствует данным |
| 5 | Сценарий "пустая неделя" работает | Ассистент предлагает начать |
| 6 | Сценарий "слабая неделя" работает | Ассистент предлагает добавить слот |
| 7 | Сценарий "хорошая неделя" работает | Ассистент хвалит и предлагает следующий шаг |

### Технические

| # | Критерий | Проверка |
|---|----------|----------|
| 1 | UI собирается без ошибок | `npm run build` успешен |
| 2 | MCP отвечает за < 3 сек | Измерение времени ответа |
| 3 | Ответы соответствуют контрактам | Валидация JSON-схемы |
| 4 | Нет console.error в UI | Проверка DevTools |

### Документация

| # | Критерий | Проверка |
|---|----------|----------|
| 1 | Бэклог заполнен | Все задачи описаны |
| 2 | Контракты MCP описаны | JSON-схемы созданы |
| 3 | Чеклист приёмки заполнен | Все пункты проверены |
| 4 | Инструкция запуска демо есть | Можно воспроизвести |
