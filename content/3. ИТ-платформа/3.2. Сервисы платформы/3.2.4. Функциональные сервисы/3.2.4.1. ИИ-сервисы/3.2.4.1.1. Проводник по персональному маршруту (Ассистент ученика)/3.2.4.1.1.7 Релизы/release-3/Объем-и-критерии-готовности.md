# Release 3: Объём и критерии готовности

## Цель релиза

Надёжность, безопасность, мониторинг — готовность к масштабированию.

---

## Что входит в релиз

### 1. Безопасность
- Rate limiting для MCP
- Отзыв токенов
- Политика удаления/экспорта данных
- Стандарты логирования без утечек (не писать заметки в логи)

### 2. Наблюдаемость
- Метрики: задержки инструментов, ошибки по типам
- Алерты: недоступна база, всплеск ошибок, рост задержек
- Dashboard мониторинга

### 3. Качество
- Автотесты сценариев (онбординг → сводка → план → итог)
- Регресс-тесты методологии

### 4. Эксплуатация
- Разделение сред (dev/staging/prod)
- Безопасные миграции схемы БД
- Бэкапы и восстановление

---

## Критерии готовности

| # | Критерий | Проверка |
|---|----------|----------|
| 1 | Rate limiting работает | При превышении лимита — 429 |
| 2 | Метрики собираются | Видны в dashboard |
| 3 | Алерты срабатывают | Тестовый инцидент → уведомление |
| 4 | Автотесты проходят | CI pipeline зелёный |
| 5 | Prod и dev разделены | Разные URL, разные данные |
