# Созидатель как создатель рабочих систем

## 0. Назначение документа

Этот документ описывает, как созидатель **создаёт рабочие системы** — команды, процессы, культуру, инструменты — внутри организации или как независимый лидер.

**Системный уровень:** Рабочие системы
**Позиция:** Созидатель как создатель целевых систем в рабочем контексте

**Методология:** Service Blueprinting, User Story Mapping, Behavioral Prototyping
**Формат:** Сценарии создания команд, процессов и культуры

---

## 1. Рабочие системы как целевые системы

### 1.1. Что такое рабочая система

Рабочая система — это устойчивая организация работы:
- **Команда** — группа людей с общей целью
- **Процесс** — последовательность действий для результата
- **Культура** — ценности и нормы поведения
- **Инструменты** — средства выполнения работы
- **Продукт** — то, что создаёт система

## 2. Создание команд

### 2.1. Жизненный цикл команды

```
ФОРМИРОВАНИЕ → ШТОРМИРОВАНИЕ → НОРМИРОВАНИЕ → ВЫПОЛНЕНИЕ → РАСФОРМИРОВАНИЕ
      │              │               │              │              │
      ▼              ▼               ▼              ▼              ▼
   Сборка        Конфликты      Правила       Результаты     Завершение
   людей         и притирка     и роли        и развитие     и наследие
```

### 2.2. Принципы формирования команды

| Принцип | Описание |
|---------|----------|
| **Цель** | Ясная общая цель, которая мотивирует |
| **Состав** | Комплементарные навыки и роли |
| **Размер** | Оптимально 5-9 человек |
| **Автономия** | Право принимать решения |
| **Ответственность** | Коллективная за результат |
| **Доверие** | Психологическая безопасность |

### 2.3. Практики командообразования

| Практика | Назначение |
|----------|------------|
| **Team charter** | Договор о целях, ролях, правилах |
| **Regular syncs** | Регулярная синхронизация |
| **Retrospectives** | Рефлексия и улучшение |
| **Team building** | Укрепление связей |
| **Shared wins** | Совместное празднование успехов |
| **Conflict resolution** | Протоколы решения конфликтов |

## 3. Создание процессов

### 3.1. Типы процессов

| Тип | Характеристика | Примеры |
|-----|----------------|---------|
| **Операционные** | Повторяющиеся, стандартизированные | Обработка заказов, поддержка |
| **Проектные** | Уникальные, ограниченные во времени | Разработка продукта |
| **Творческие** | Нелинейные, итеративные | Дизайн, исследования |
| **Управленческие** | Принятие решений, контроль | Планирование, найм |

### 3.2. Дизайн процесса

| Элемент | Вопросы |
|---------|---------|
| **Вход** | Что запускает процесс? |
| **Шаги** | Какие действия и в какой последовательности? |
| **Роли** | Кто выполняет каждый шаг? |
| **Решения** | Где точки принятия решений? |
| **Артефакты** | Что создаётся на каждом этапе? |
| **Выход** | Какой конечный результат? |
| **Метрики** | Как измерять эффективность? |

### 3.3. Оптимизация процессов

Созидатель применяет:
- **Lean** — устранение потерь
- **Six Sigma** — снижение вариативности
- **Theory of Constraints** — расшивка узких мест
- **Automation** — автоматизация рутины
- **Continuous Improvement** — постоянные улучшения

## 6. Типичные сценарии

### 6.1. Создание новой команды

**Контекст:** Запуск нового проекта/продукта.

**Что делает созидатель:**
1. Определяет цель и scope команды
2. Формирует состав (навыки, роли)
3. Проводит kick-off (цели, правила, ожидания)
4. Устанавливает ритмы (standup, retro)
5. Создаёт инфраструктуру (каналы, доски, репо)
6. Выстраивает культуру через первые месяцы

### 6.2. Трансформация существующего процесса

**Контекст:** Неэффективный процесс требует изменений.

**Что делает созидатель:**
1. Изучает текущий процесс (as-is mapping)
2. Идентифицирует проблемы и bottlenecks
3. Проектирует целевой процесс (to-be)
4. Вовлекает stakeholders
5. Пилотирует изменения
6. Масштабирует при успехе
7. Мониторит и корректирует

### 6.3. Изменение культуры подразделения

**Контекст:** Токсичная или неэффективная культура.

**Что делает созидатель:**
1. Диагностирует текущую культуру
2. Определяет целевую культуру
3. Согласовывает изменения с руководством
4. Начинает с себя (личный пример)
5. Меняет ритуалы и практики
6. Награждает правильное поведение
7. Расстаётся с toxic людьми
8. Терпеливо ждёт (культура меняется медленно)

## 7. Инструменты и методы

### 7.1. Для команд

| Инструмент | Назначение |
|------------|------------|
| **Scrum/Kanban** | Организация работы |
| **OKR** | Целеполагание |
| **RACI** | Распределение ответственности |
| **Team Health Check** | Диагностика состояния |

### 7.2. Для процессов

| Инструмент | Назначение |
|------------|------------|
| **BPMN** | Моделирование процессов |
| **Value Stream Mapping** | Анализ потока создания ценности |
| **Swimlane Diagrams** | Визуализация ролей в процессе |

## 8. Антипаттерны

| Антипаттерн | Проблема | Альтернатива |
|-------------|----------|--------------|
| **Overprocessing** | Слишком много правил | Минимальные необходимые процессы |
| **Hero culture** | Зависимость от отдельных людей | Распределённая ответственность |
| **Blame game** | Поиск виноватых | Культура обучения |
| **Micromanagement** | Контроль каждого шага | Доверие и автономия |
| **Change fatigue** | Постоянные реорганизации | Стабильность с точечными изменениями |
