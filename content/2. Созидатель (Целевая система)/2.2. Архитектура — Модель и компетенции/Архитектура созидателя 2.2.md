# Архитектура созидателя

## 0. Назначение документа

Этот документ описывает **полную архитектуру созидателя** — системную модель человека как целевой системы экосистемы развития. Архитектура объединяет функциональное разбиение, модульную структуру, характеристики, состояния, роли и траектории развития в единое целое.

Документ следует принципам FPF (First Principles Framework) и служит навигационным центром для всех архитектурных описаний раздела 2.2.

---

## 1. Созидатель как система (FPF A.1 Holonic)

### 1.1. Определение

**Созидатель** — это человек, развивающийся в экосистеме и создающий ценность для мира. В терминах FPF:

| Аспект | Описание |
|--------|----------|
| **Как целое** | Автономный агент с собственными целями, интересами, траекторией |
| **Как часть** | Элемент надсистем: семьи, организации, сообщества, цивилизации |
| **Как холон** | Одновременно целое и часть, с эмерджентными свойствами |

### 1.2. Системный контекст

```
┌─────────────────────────────────────────────────────────────────┐
│                         НАДСИСТЕМА (1.)                         │
│  Мир: семья, организации, сообщества, общество, цивилизация    │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ интерфейсы
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                    ЦЕЛЕВАЯ СИСТЕМА (2.)                         │
│                        СОЗИДАТЕЛЬ                               │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────────┐ │
│  │  Организм   │  │  Личность   │  │       Экзотело          │ │
│  │  (тело)     │◄─┤  (разум)    │◄─┤ (внешние расширения)    │ │
│  └─────────────┘  └─────────────┘  └─────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ поддержка развития
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                   СИСТЕМА СОЗДАНИЯ (3.)                         │
│  Экосистема: платформа, программы, наставники, ИИ-агенты       │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3. Трансформация

Экосистема создаёт и развивает созидателя через **трансформацию**:

```
Вход (αS)                    Трансформация (T)                   Выход (ωS)
┌──────────────┐             ┌────────────────┐              ┌──────────────┐
│ Человек      │ ──────────► │ Программы      │ ──────────►  │ Созидатель   │
│ (потенциал)  │             │ развития       │              │ (мастерство) │
└──────────────┘             └────────────────┘              └──────────────┘
```

## 3. Функциональное разбиение

### 3.1. Два уровня функций

```
Созидатель (агент)
├── ЛИЧНОСТЬ (информационная подсистема)
│   ├── Мыслительное мастерство
│   │   ├── Мотивационная подсистема      → зачем действовать
│   │   ├── Аналитическая подсистема      → понимание ситуации
│   │   ├── Планировочная подсистема      → что и как делать
│   │   └── Контрольная подсистема        → обратная связь
│   └── Прикладное мастерство
│       └── Профессиональные методы       → как делать хорошо
│
└── ОРГАНИЗМ (физическая подсистема)
    ├── Сенсорная система                 → восприятие мира
    ├── Моторная система                  → действие в мире
    └── Энергетическая система            → поддержание жизни
```

### 3.2. Функции личности

| Подсистема | Функция | Ключевые методы |
|------------|---------|-----------------|
| **Мотивационная** | Целеполагание, смыслообразование | Стратегирование, рефлексия |
| **Аналитическая** | Анализ, прогнозирование | Моделирование, логика |
| **Планировочная** | Декомпозиция, приоритизация | Планирование, тайм-менеджмент |
| **Контрольная** | Мониторинг, коррекция | Обратная связь, итерации |
| **Прикладная** | Исполнение методов | Профессиональные навыки |

### 3.3. Функции организма

| Подсистема | Функция | Критические показатели |
|------------|---------|------------------------|
| **Сенсорная** | Получение информации | Внимание, восприятие |
| **Моторная** | Преобразование мира | Координация, выносливость |
| **Энергетическая** | Поддержание работы | Сон, питание, восстановление |

## 4. Модульное разбиение: киберчеловек

### 4.1. Структура киберчеловека

Современный созидатель — это **киберчеловек**: биологическое тело + внешние расширения (экзотело).

```
Киберчеловек
├── БИОЛОГИЧЕСКОЕ ТЕЛО
│   ├── Мозг (нейронные сети)
│   ├── Тело (мышцы, органы)
│   └── Сенсоры (органы чувств)
│
└── ЭКЗОТЕЛО (внешние расширения)
    ├── ЭКЗОКОРТЕКС (когнитивные расширения)
    │   ├── Заметочники (second brain)
    │   ├── Базы знаний
    │   ├── Цифровой двойник
    │   └── ИИ-агенты и ассистенты
    │
    └── ИСПОЛНИТЕЛЬНЫЕ РАСШИРЕНИЯ
        ├── Инструменты и устройства
        ├── Финансовые средства
        └── Имущество и инфраструктура
```

### 4.2. Экзокортекс

**Экзокортекс** — когнитивные расширения, усиливающие мышление:

| Компонент | Назначение | Примеры |
|-----------|------------|---------|
| **Заметочники** | Внешняя память, связывание идей | Obsidian, Notion, Roam |
| **Базы знаний** | Структурированное хранение | Wiki, репозитории |
| **Цифровой двойник** | Отслеживание развития | Показатели, траектория |
| **ИИ-агенты** | Расширение интеллекта | ChatGPT, Claude, специализированные агенты |

### 4.3. Интеграция человек-ИИ

Ключевая характеристика современного созидателя — **техноинтеграция**: способность эффективно работать в связке с ИИ.

| Уровень | Описание | Компетенции |
|---------|----------|-------------|
| 1 | Пользователь | Использует готовые инструменты |
| 2 | Настройщик | Адаптирует инструменты под задачи |
| 3 | Оркестратор | Управляет связками инструментов |
| 4 | Архитектор | Проектирует персональные системы |
| 5 | Со-создатель | Партнёрство человек-ИИ |

**Подробнее:** [[Экзотело]], [[Интеграция человек-ИИ]], [[Техноинтеграция]]

## 5. Характеристики созидателя

### 5.1. Модель характеристик (FPF A.17-A.18)

Характеристики — это **измеримые свойства** созидателя. Каждая характеристика описывается по стандарту CSLC:

- **C**haracteristic — название
- **S**cale — тип шкалы
- **L**evels — уровни/градации
- **C**oordinates — якоря шкалы

### 5.2. Шесть киберхарактеристик

Ключевые характеристики для инженерного проектирования развития:

| # | Характеристика | Описание | Шкала |
|---|----------------|----------|-------|
| 1 | **Агентность** | Способность быть источником действия | 1-5 |
| 2 | **Интеллектуальная развитость** | Качество мышления и познания | 1-5 |
| 3 | **Стрессоустойчивость** | Устойчивость под давлением | 1-5 |
| 4 | **Ресурсность** | Доступ к времени, финансам, связям | 1-5 |
| 5 | **Техноинтеграция** | Интеграция с экзотелом | 1-5 |
| 6 | **Калибр личности** | Масштаб замыслов и влияния | 1-5 |

### 5.3. Дополнительные группы характеристик

| Группа | Количество | Примеры |
|--------|------------|---------|
| Физические | 5 | Выносливость, сила, гибкость |
| Когнитивные | 5 | Обучаемость, креативность, рациональность |
| Социальные | 5 | Коммуникативность, лидерство, эмпатия |
| Волевые | 5 | Самодисциплина, настойчивость, решительность |
| Интегральные | 4 | Потенциал, антихрупкость, эволюционируемость |

**Общая размерность CHR-Space:** 26 измерений

### 5.4. CHR-Space: пространство характеристик

```
CHR-Space = (D, S, T)

где:
D = {d₁, d₂, ..., d₂₆}  — множество измерений (характеристик)
S = D₁ × D₂ × ... × D₂₆ — пространство состояний
T: Time → S             — траектория развития
```

- **Состояние** созидателя — точка в CHR-Space
- **Развитие** — траектория движения точки
- **Цель** — целевое состояние (role anchor)

## 6. Состояния созидателя

### 6.1. Концепция состояний

**Состояние** = совокупность текущих значений показателей характеристик.

```
Характеристики → Показатели → Значения → Состояние
```

| Категория | Описание | Примеры |

| **Продуктивное** | Готовность к работе | Фокус, поток, мобилизация |

| **Социальное** | Включённость | Изоляция, активность, лидерство |

## 7. Ролевая траектория

### 7.1. Пять ключевых ролей

Созидатель проходит через последовательность ролей:

```
Ученик → Интеллектуал → Профессионал → Исследователь → Просветитель
   ↓           ↓              ↓               ↓              ↓
 Личное    Понятийная     Прикладное     Исследоват.    Публичное
развитие    работа        мастерство      мастерство    влияние
```

| Роль | Основная функция | Ключевое мастерство | Программа |
|------|------------------|---------------------|-----------|
| **Ученик** | Освоение нового | Практики саморазвития | Личное развитие |
| **Интеллектуал** | Мышление, понятия | Мыслительное мастерство | Рабочее развитие |
| **Профессионал** | Создание продуктов | Прикладное мастерство | Дополнительные курсы |
| **Исследователь** | Создание нового | Исследовательское мастерство | Исследовательское развитие |
| **Просветитель** | Передача знаний | Просветительское мастерство | Просветительское развитие |

### 7.2. Ступени роли Ученика

Внутри роли Ученика выделяются поведенческие ступени:

| Ступень | Описание | Маркеры |
|---------|----------|---------|
| **Случайный** | Нерегулярное обучение | Эпизодические слоты |
| **Практикующий** | Регулярные практики | 3+ слота в неделю |
| **Систематический** | Устойчивый ритм | Недельный контракт |
| **Дисциплинированный** | 80%+ выполнение | 10+ часов в неделю |
| **Проактивный** | Инициирует развитие | Создаёт свои практики |

## 12. Архитектурные принципы (FPF)

### 12.1. Холонический принцип (A.1)

Созидатель — холон: одновременно целое (автономный агент) и часть (элемент надсистем).

### 12.2. Строгое различение (A.7)

| Сущность | Не путать с |
|----------|-------------|
| Роль | Должность, профессия |
| Характеристика | Состояние |
| Метод | Инструмент |
| Документ | Система |

### 12.3. Trust Calculus (B.3)

Доверие к архитектурным описаниям:

```
Trust = ⟨F, G, R⟩

F = Формальность описания
G = Область применимости
R = Надёжность (проверенность)
```

### 12.4. ADI-цикл (B.5)

При проектировании развития:

1. **Abduction** — гипотезы о траектории
2. **Deduction** — проверка логики
