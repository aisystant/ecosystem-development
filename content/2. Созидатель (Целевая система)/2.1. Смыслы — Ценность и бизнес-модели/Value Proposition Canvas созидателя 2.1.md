# Value Proposition Canvas созидателя

## 0. Назначение документа

Этот документ обосновывает, **почему созидатель является решением проблем**, описанных в [[Проблемы и вызовы мира 1.1]]. Документ построен по методологии Value Proposition Canvas и отвечает на вопрос: «Почему человек примет решение стать созидателем?»

**Связь с разделом 1.1:** Проблемы мира фиксируют «боли» и «задачи» аудитории. Этот документ показывает, как созидатель их решает.

---

## 1. Профиль клиента (Customer Profile)

### 1.1. Задачи клиента (Customer Jobs)

**Функциональные задачи:**
| Задача | Описание | Источник проблемы |
|--------|----------|-------------------|
| Построить личную траекторию развития | Иметь понятный маршрут роста, а не хаотичные курсы | Проблема 1: Свобода без «грамматики» |
| Развить системное мышление | Уметь осмыслять сложное, моделировать, переносить знания | Проблема 2: Интеллект как узкое место |
| Создавать ценные артефакты | Производить результаты, а не потреблять контент | Проблема 4: Мемы классического обучения |
| Встроить обучение в работу | Учиться внутри производства, а не «рядом» | Проблема 7: Обучение рядом с работой |
| Грамотно использовать ИИ | Применять ИИ как инструмент мышления, а не «автопилот» | Проблема 8: Стихийное использование ИИ |
| Иметь подтверждённое мастерство | Доказательства навыков через артефакты, не дипломы | Проблема 10: Рынок сигналов |

**Социальные задачи:**
- Быть признанным профессионалом в своей области
- Входить в сообщество системных мыслителей
- Влиять на развитие своей сферы деятельности

**Эмоциональные задачи:**
- Чувствовать прогресс и осмысленность развития
- Уверенность в своих решениях
- Удовлетворение от созданных результатов

---

### 1.2. Боли клиента (Pains)

| Боль | Интенсивность | Связь с проблемами мира |
|------|---------------|-------------------------|
| «Не понимаю, чему учиться и в каком порядке» | Высокая | Проблема 1 |
| «Прохожу курсы, но знания не применяются» | Высокая | Проблема 4, 7 |
| «Не хватает времени на фундаментальное развитие» | Критическая | Проблема 3 |
| «ИИ делает за меня, но я не понимаю, что происходит» | Средняя | Проблема 8, 16 |
| «Сертификаты не помогают в реальной работе» | Высокая | Проблема 10 |
| «После университета развитие остановилось» | Средняя | Проблема 12 |
| «Учусь один, нет поддержки и обратной связи» | Средняя | Проблема 6 |
| «Не могу объяснить свою ценность работодателю» | Высокая | Проблема 10 |

**Барьеры:**
- Нет времени — работа поглощает всё внимание
- Нет ритма — попытки срываются через 2-3 недели
- Нет окружения — близкие не понимают, зачем это нужно
- Нет критериев — непонятно, прогрессирую или нет

---

### 1.3. Выгоды клиента (Gains)

| Выгода | Приоритет | Ожидаемый результат |
|--------|-----------|---------------------|
| Понятный маршрут развития | Критический | Знаю следующий шаг и вижу горизонт |
| Практические навыки | Критический | Применяю новое в работе уже на 2-й неделе |
| Рабочие артефакты | Высокий | Портфолио из реальных проектов |
| Устойчивый ритм | Высокий | Недельные слоты становятся привычкой |
| Сообщество практиков | Средний | Peer review и наставничество |
| Признанное мастерство | Средний | Репутация подтверждается артефактами |
| Рост дохода | Высокий | Компетенции конвертируются в оплату |

### 2.2. Обезболиватели (Pain Relievers)

| Боль | Как созидатель решает |
|------|----------------------|
| «Не понимаю, чему учиться» | Персональный маршрут с диагностикой и приоритизацией |
| «Знания не применяются» | Обучение через создание артефактов в реальных проектах |
| «Нет времени на развитие» | Недельные слоты по 2-4 часа встроены в рабочий ритм |
| «ИИ делает за меня» | Экзокортекс учит думать с ИИ, а не вместо ИИ |
| «Сертификаты бесполезны» | Портфолио артефактов вместо бумажных подтверждений |
| «Учусь один» | Сообщество, наставники, еженедельный peer review |
| «Не могу доказать ценность» | Цифровой двойник с историей артефактов и репутацией |

---

### 2.3. Создатели выгод (Gain Creators)

| Желаемая выгода | Как созидатель создаёт |
|-----------------|----------------------|
| Понятный маршрут | Траектория Ученик→Интеллектуал→Профессионал→Исследователь→Просветитель |
| Практические навыки | Каждая неделя = артефакт, применимый в работе |
| Рабочие артефакты | Стажировки и резидентуры на реальных проектах |
| Устойчивый ритм | Недельный цикл: слот→инкремент→review→перенос |
| Сообщество | Peer группы, наставники, совместные проекты |
| Признание | Токены за вклад, публичная репутация |
| Рост дохода | Компетенции → проекты → доход (модель в [[Business Model Canvas созидателя 2.1]]) |

## 3. Fit — соответствие профиля и ценности

### 3.1. Карта соответствия

```
ПРОБЛЕМЫ МИРА (1.1)          ЗАДАЧИ КЛИЕНТА           РЕШЕНИЕ СОЗИДАТЕЛЯ
─────────────────────────────────────────────────────────────────────────
Свобода без грамматики   →   Личная траектория    →   Персональный маршрут
Интеллект как узкое      →   Системное мышление   →   Руководства + практика
Культ быстрых побед      →   Устойчивый ритм      →   Недельные слоты
Мемы классического       →   Реальные навыки      →   Артефакты, не дипломы
Обучение рядом           →   Учиться в работе     →   Стажировки + резидентуры
Стихийное ИИ             →   Грамотный ИИ         →   Экзокортекс
Рынок сигналов           →   Доказать мастерство  →   Цифровой двойник
```

### 3.2. Уникальность позиционирования

**Созидатель ≠ «выпускник курсов»**

| Традиционное обучение | Созидатель |
|----------------------|------------|
| Проходит курс | Создаёт артефакты |
| Получает сертификат | Накапливает портфолио |
| Учится отдельно от работы | Учится внутри работы |
| Один раз «прошёл» | Непрерывное развитие |
| Зависит от преподавателя | Автономен с поддержкой |
| Использует ИИ стихийно | Мыслит с ИИ системно |

## 4. Почему созидатель — единственное решение?

### 4.1. Альтернативы и их ограничения

| Альтернатива | Что решает | Что НЕ решает |
|--------------|------------|---------------|
| Онлайн-курсы | Доступ к контенту | Применение, ритм, перенос |
| MBA/магистратура | Сигнал статуса | Практические навыки, время |
| Менторство | Индивидуальность | Масштаб, системность |
| Самообучение | Гибкость | Структура, сообщество, проверка |
| Корпоративное обучение | Бесплатность | Личная траектория, глубина |

### 4.2. Уникальная комбинация созидателя

Созидатель объединяет то, что обычно несовместимо:
- **Персонализация** + **Масштабируемость** (через цифровой двойник)
- **Глубина** + **Практичность** (руководства + стажировки)
- **Автономия** + **Поддержка** (ритмы + сообщество)
- **Мышление** + **Инструменты** (методы + ИИ-ассистенты)

## 5. Ключевые гипотезы для валидации

| # | Гипотеза | Метрика проверки | Статус |
|---|----------|------------------|--------|
| 1 | Люди готовы платить за персональный маршрут развития | Конверсия в первый платёж >5% | Pending |
| 2 | Недельный ритм удерживает лучше, чем курсы | Retention 12 недель >40% | Pending |
| 3 | Артефакты ценнее сертификатов для работодателей | NPS от работодателей >50 | Pending |
| 4 | ИИ-ассистенты ускоряют освоение в 2-3 раза | Время до первого артефакта <2 недели | Pending |
