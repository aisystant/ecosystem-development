# Глоссарий хранилища 0.1

## 0. Назначение документа

Этот документ содержит определения ключевых терминов, используемых в хранилище экосистемы. Глоссарий служит единым источником истины для терминологии и обеспечивает согласованность языка между документами.

---

## 1. Термины структуры хранилища

| Термин | Определение |
|--------|-------------|
| **Хранилище** | Git-репозиторий с документацией экосистемы, организованный по системной модели |
| **Раздел** | Папка верхнего уровня (0, 1, 2, 3), соответствующая уровню системной модели |
| **Подраздел** | Папка второго уровня (N.M), соответствующая роли мышления |
| **Подподраздел** | Папка третьего уровня (N.M.K) для группировки связанных документов |
| **Ячейка** | Место в структуре хранилища на пересечении уровня системы и роли мышления |
| **О разделе** | Информационный документ, описывающий назначение и содержание раздела/подраздела |

---

## 2. Термины системной модели

| Термин | Определение |
|--------|-------------|
| **Надсистема (Мир)** | Внешний контекст, в который встраивается созидатель: рынок, аудитории, партнёры |
| **Целевая система (Созидатель)** | Человек, которого развивает экосистема — основной «продукт» системы |
| **Система создания (Экосистема)** | Платформа, команда и процессы, создающие и развивающие созидателей |
| **Трансформация** | Процесс развития человека в созидателя через прохождение программ экосистемы |

---

## 3. Термины ролей мышления

| Термин | Определение |
|--------|-------------|
| **Предприниматель** | Роль мышления, фокусирующаяся на смыслах и ценности. Главный вопрос: «Зачем?» |
| **Инженер** | Роль мышления, фокусирующаяся на архитектуре и устройстве. Главный вопрос: «Как устроено?» |
| **Менеджер** | Роль мышления, фокусирующаяся на операциях и процессах. Главный вопрос: «Как работает?» |

---

## 4. Термины семейств документов

| Термин | Определение |
|--------|-------------|
| **Семейство документов** | Группа документов F0-F9, объединённых по системному уровню и роли мышления |
| **F0** | Метасистема — управление знаниями и правила хранилища |
| **F1** | Надсистема × Предприниматель — видение и привлечение |
| **F2** | Надсистема × Инженер — контекст и интерфейсы |
| **F3** | Надсистема × Менеджер — репутация и партнёрства |
| **F4** | Целевая × Предприниматель — ценность и бизнес-модели |
| **F5** | Целевая × Инженер — модель и компетенции |
| **F6** | Целевая × Менеджер — путь и сопровождение |
| **F7** | Создания × Предприниматель — экономика и инвестиции |
| **F8** | Создания × Инженер — платформа и подсистемы |
| **F9** | Создания × Менеджер — команда и службы |

---

## 5. Термины документов и метаданных

| Термин | Определение |
|--------|-------------|
| **Frontmatter** | YAML-блок в начале документа с метаданными (type, status, created, layer, scope) |
| **ADR** | Architecture Decision Record — запись о принятом архитектурном решении с контекстом, решением и последствиями |
| **SSOT** | Single Source of Truth — принцип единственного источника истины для каждой сущности |
| **Шаблон** | Стандартная структура документа для обеспечения единообразия |
| **Статус документа** | Уровень зрелости: draft (черновик), active (рабочий), approved (утверждённый), archived (архивный) |

---

## 6. Термины процессов

| Термин | Определение |
|--------|-------------|
| **Docs-as-Code** | Подход к документации как к коду: версионирование, PR, code review |
| **PR (Pull Request)** | Запрос на слияние изменений в основную ветку через ревью |
| **Owner** | Ответственный за актуальность документа |
| **Maintainer** | Участник, имеющий право принимать PR в документ |
| **Contributor** | Участник, имеющий право предлагать изменения через PR |

---

## 7. Термины экосистемы

| Термин | Определение |
|--------|-------------|
| **Созидатель** | Человек, развивающийся в экосистеме и создающий ценность для мира |
| **Экосистема развития** | Совокупность платформы, команды, процессов и ИИ-ассистентов для развития созидателей |
| **ИИ-ассистент** | Автономный агент на базе LLM, выполняющий задачи в экосистеме |
| **Платформа** | Технологическая инфраструктура экосистемы (ИТ-системы, API, интеграции) |
| **Служба** | Организационная единица экосистемы (Школа, Мастерская, Лаборатории, Клуб, Деканат) |
| **Токеномика** | Экономическая модель экосистемы на основе внутренних токенов |
| **Proof-of-Impact** | Система доказательства вклада участника в экосистему |
| **Цифровой двойник** | Цифровое представление созидателя: компетенции, прогресс, цели |
| **Персональный маршрут** | Индивидуальная траектория развития созидателя |
| **Деканат** | Служба сопровождения обучающихся в экосистеме |

---

## 8. Термины ИИ-агентов

| Термин | Определение |
|--------|-------------|
| **Orchestrator** | ИИ-агент, управляющий другими агентами и координирующий их работу |
| **ConsistencyChecker** | ИИ-агент, проверяющий противоречия и несогласованности в документах |
| **Librarian** | ИИ-агент, помогающий искать и классифицировать документы |
| **SystemArchitect** | ИИ-агент, помогающий с архитектурными решениями |
| **Паспорт агента** | Документ, описывающий роль, входы, выходы и метрики качества ИИ-агента |

---

## 9. Система обозначений сущностей

В документах экосистемы используется единая система обозначений с префиксами:

| Префикс | Семейство | Примеры |
|---------|-----------|---------|
| `R.*` | Роли | R.Student, R.GrowthGuide |
| `S.*` | Системы | S.LMS, S.AIPlatform |
| `C.*` | Образовательные сущности | C.Program.*, C.Guide.* |
| `AGT.*` | ИИ-агенты | AGT.RouteGuide, AGT.Navigator |
| `IND.*`, `MET.*` | Показатели и метрики | IND.1.3.4, MET.2.1.1 |

Полный перечень семейств и правила именования см. в [[Словарь сущностей и правила обозначений для экосистемы развития]].
