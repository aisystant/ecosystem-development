# Глоссарий хранилища 0.1

## 0. Назначение документа

Этот документ содержит определения ключевых терминов, используемых в хранилище экосистемы. Глоссарий служит единым источником истины для терминологии и обеспечивает согласованность языка между документами.

---

## 1. Термины структуры хранилища

| Термин | Определение |
|--------|-------------|
| **Хранилище** | Git-репозиторий с документацией экосистемы, организованный по системной модели |
| **Раздел** | Папка верхнего уровня (0, 1, 2, 3), соответствующая уровню системной модели |
| **Подраздел** | Папка второго уровня (N.M), соответствующая роли мышления |
| **Подподраздел** | Папка третьего уровня (N.M.K) для группировки связанных документов |
| **Ячейка** | Место в структуре хранилища на пересечении уровня системы и роли мышления |
| **О разделе** | Информационный документ, описывающий назначение и содержание раздела/подраздела |

---

## 2. Термины системной модели

| Термин | Определение |
|--------|-------------|
| **Надсистема (Мир)** | Внешний контекст, в который встраивается созидатель: рынок, аудитории, партнёры |
| **Целевая система (Созидатель)** | Человек, которого развивает экосистема — основной «продукт» системы |
| **Система создания (Экосистема)** | Платформа, команда и процессы, создающие и развивающие созидателей |
| **Трансформация** | Процесс развития человека в созидателя через прохождение программ экосистемы |

---

## 3. Термины ролей мышления

| Термин | Определение |
|--------|-------------|
| **Предприниматель** | Роль мышления, фокусирующаяся на смыслах и ценности. Главный вопрос: «Зачем?» |
| **Инженер** | Роль мышления, фокусирующаяся на архитектуре и устройстве. Главный вопрос: «Как устроено?» |
| **Менеджер** | Роль мышления, фокусирующаяся на операциях и процессах. Главный вопрос: «Как работает?» |

## 4.1. Термины целевых ролей документов

| Термин | Определение |
|--------|-------------|
| **Целевая аудитория** | Группа людей или ИИ-агентов, для которых предназначен документ. Определяется семейством документа (F0-F9) |

| **Ошибка классификации** | Несоответствие между целевой аудиторией документа и его семейством. Критическая ошибка для ИИ-анализа |

## 5. Термины документов и метаданных

| Термин | Определение |
|--------|-------------|
| **Frontmatter** | YAML-блок в начале документа с метаданными (type, status, created, layer, scope) |
| **ADR** | Architecture Decision Record — запись о принятом архитектурном решении с контекстом, решением и последствиями |
| **SSOT** | Single Source of Truth — принцип единственного источника истины для каждой сущности |
| **Шаблон** | Стандартная структура документа для обеспечения единообразия |
| **Статус документа** | Уровень зрелости: draft (черновик), active (рабочий), approved (утверждённый), archived (архивный) |

## 7. Термины экосистемы

| Термин | Определение |
|--------|-------------|
| **Созидатель** | Человек, развивающийся в экосистеме и создающий ценность для мира |
| **Экосистема развития** | Совокупность платформы, команды, процессов и ИИ-ассистентов для развития созидателей |
| **ИИ-ассистент** | Система на базе LLM, оказывающая помощь пользователю по запросу (объясняет, проверяет, рекомендует, редактирует, сопровождает). В отличие от ИИ-агента работает реактивно, а не автономно |
| **ИИ-агент** | Система на базе LLM с целевой автономностью: планирует шаги, вызывает инструменты, выполняет цепочки действий, ведёт состояние/память |
| **ИТ-платформа Aisystant** | Технологическая инфраструктура экосистемы (ИТ-системы, API, интеграции, ИИ-компоненты). Синонимы: ИИ-платформа Aisystant, ИТ-платформа, ИИ-платформа, платформа Aisystant |
| **Служба** | Организационная единица экосистемы (Школа, Мастерская, Лаборатории, Клуб, Деканат) |
| **Токеномика** | Экономическая модель экосистемы на основе внутренних токенов |
| **Proof-of-Impact** | Proof-of-Impact участника в экосистему |
| **Цифровой двойник** | Цифровое представление созидателя: компетенции, прогресс, цели |
| **Персональный маршрут** | Индивидуальная траектория развития созидателя |
| **Деканат** | Служба сопровождения обучающихся в экосистеме |
| **Ассистент Ученика** (Learning Copilot) | ИИ-ассистент «Проводник по персональному маршруту» — помогает взрослому человеку пройти путь от Случайного до Проактивного ученика, опираясь на методику роли Ученика и данные цифрового двойника |

## 9. Система обозначений сущностей

В документах экосистемы используется единая система обозначений с префиксами:

| Префикс | Семейство | Примеры |
|---------|-----------|---------|
| `R.*` | Роли | R.Student, R.GrowthGuide |
| `S.*` | Системы | S.LMS, S.AIPlatform |
| `C.*` | Образовательные сущности | C.Program.*, C.Guide.* |
| `AGT.*` | ИИ-агенты | AGT.RouteGuide, AGT.Navigator |
| `IND.*`, `MET.*` | Показатели и метрики | IND.1.3.4, MET.2.1.1 |

Полный перечень семейств и правила именования см. в [[Словарь сущностей и правила обозначений для экосистемы развития]].

## 10. Термины классификации систем и ИИ

### 10.1. Базовые понятия

| Термин | Определение |
|--------|-------------|
| **Система** | Целевая или обеспечивающая сущность (организационная, социотехническая, продуктовая, цифровая и т.п.), имеющая назначение, границы, интерфейсы и результаты |
| **ИТ-система** | Цифровая (программно-техническая) система: код, данные, интеграции, интерфейсы, эксплуатационные контуры (наблюдаемость, безопасность, CI/CD и т.д.). ИТ-система может не содержать ИИ или содержать его как опциональную функцию |
| **Детерминированная ИТ-система** | ИТ-система, в которой ключевые функции реализованы явными алгоритмами/правилами (предсказуемое поведение при одинаковых входах). ИИ (если используется) не является смысловым ядром и может быть отключён/заменён без разрушения основной функции |
| **ИИ-система** | ИТ-система, у которой смысловая функция на уровне требований достигается через инференс модели: система "выводит", как из входов получить выходы (например, предсказания, контент, рекомендации, решения), и может иметь уровни автономности/адаптивности после развёртывания. Источники: [OECD AI Principles](https://oecd.ai), [NIST AI RMF](https://nvlpubs.nist.gov) |
| **ИИ-ассистент** | Подтип ИИ-системы с диалоговым/помогающим интерфейсом и контрактом "помощь пользователю" (объясняет, проверяет, рекомендует, редактирует, сопровождает) |
| **ИИ-агент** | Подтип ИИ-системы с целевой автономностью: планирует шаги, вызывает инструменты, выполняет цепочки действий, ведёт состояние/память и следует политикам |

### 10.2. Нормативное правило: классификация по роли ИИ

Для каждого компонента экосистемы фиксируется метка `ai_role` в frontmatter документа:

| Метка | Значение | Применение | Примеры |
|-------|----------|------------|---------|
| `ai_role: none` | Детерминированная ИТ-система | ИИ отсутствует | CRM, LMS, биллинг, системы оплаты |
| `ai_role: feature` | ИТ-система с ИИ-фичей | ИИ улучшает функциональность, но не является ядром | Клуб с AI-рекомендациями, LMS с AI-подсказками |
| `ai_role: core` | ИИ-система | Без инференса система не выполняет основную функцию | ИИ-ассистенты, ИИ-агенты, эпистемический граф |

**Следствия для ИИ-систем** (`ai_role: core`):
- Обязательны контуры оценки качества модели
- Требуется мониторинг поведения модели
- Необходимо управление рисками ИИ
- Где требуется: интеграция "человек-в-контуре" (human-in-the-loop)

**Принцип именования:**
- Детерминированные ИТ-системы размещаются в разделе "Детерминированные ИТ-системы"
- ИИ-системы (ассистенты и агенты) размещаются в соответствующих разделах "ИИ-ассистенты" и "ИИ-агенты"
