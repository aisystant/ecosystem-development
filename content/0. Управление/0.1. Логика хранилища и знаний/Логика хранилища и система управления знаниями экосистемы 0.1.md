
Документ написан так, чтобы:  
— объяснять структуру хранилища человеку;  
— задавать правила работы для ИИ;  
— обеспечивать непротиворечивость и полноту 3×3;  
— позволять автоматически классифицировать документы.

## 1. Назначение документа

Этот документ описывает единую логику организации хранилища знаний экосистемы, включая:

1. Принципы структуры репозитория (0–3 уровни).
    
2. Логику таблицы 3×3 как основы классификации документов.
    
3. Правила автоматической работы ИИ-ассистентов с документами:  
    — определение места для новых документов;  
    — выявление пробелов и отсутствующих элементов;  
    — предложения по актуализации документов;  
    — проверка непротиворечивости.
    
4. Описание технического слоя уровня 0, в котором размещаются документы, автоматически создаваемые ИИ для анализа качества хранилища.
    

Документ является базовым для всей архитектуры знаний экосистемы.

---

## 2. Принципы организации хранилища

### 2.1 Минимализм уровней

Верхнеуровневая структура состоит из четырёх разделов:

```
0. Управление
1. Мир
2. Созидатель
3. Экосистема
```

Разделы 1–3 имеют строго одинаковую внутреннюю структуру:

```
X.1 Смыслы (предпринимательский взгляд)
X.2 Архитектура (инженерный взгляд)
X.3 Операции (менеджерский взгляд)
```

Где X = 1, 2 или 3.

### 2.2 Логика таблицы 3×3

Таблица 3×3 определяет _семейства документов_. Каждая ячейка — это уникальный «тип мышления» и формат содержания.

Три строки — три системы:

1. **Мир** — надсистема и запрос реальности.
    
2. **Созидатель** — система создания.
    
3. **Экосистема** — целевая система.
    

Три столбца — три роли:

1. **Предприниматель** (смыслы, ценность, востребованность).
    
2. **Инженер** (устройство, модели, архитектура).
    
3. **Менеджер** (операции, процессы, эксплуатация).
    

В каждой ячейке должен находиться главный документ, который описывает:

— цель ячейки,  
— содержание,  
— формат мышления,  
— перечень обязательных рабочих продуктов.

### 2.3 Принцип единственного места

Каждый документ должен иметь одно-единственное правильное место в хранилище.

Это обеспечивает:

— непротиворечивость;  
— отсутствие дубликатов;  
— однозначность интерпретации.

### 2.4 Принцип связности

Каждый документ должен указывать:

— на документы уровня выше (контекст);  
— на документы уровня ниже (детализация);  
— на документы той же строки или столбца (пересечения смыслов, архитектур и операций).

---

## 3. Правила создания документов

### 3.1 Каждый документ относится к ровно одной ячейке 3×3

Документ классифицируется по следующим осям:

1. Что является описываемой системой?  
    — Мир  
    — Созидатель  
    — Экосистема
    
2. Какой тип мышления используется?  
    — Смысловой/ценностный  
    — Архитектурный  
    — Операционный
    

ИИ-ассистент должен автоматически определять эти признаки, анализируя:

— ключевые слова;  
— структуру документа;  
— описываемые объекты;  
— тип аргументации;  
— рабочие продукты.

### 3.2 Каждый документ обязан иметь:

1. Цель.
    
2. Описание контекста.
    
3. Основной текст.
    
4. Ссылки на соседние ячейки.
    
5. Рабочие продукты (если применимо).
    
6. Версию и дату изменения.
    

### 3.3 Документы в пределах одной ячейки

Каждая ячейка содержит:

1. **Главный документ ячейки** — основной.
    
2. **Развивающие документы** — узкие темы, уточнения, схемы.
    
3. **Шаблоны рабочих продуктов** для этой ячейки.
    

---

## 4. Работа ИИ-ассистента с документами

ИИ должен уметь:

### 4.1 Определять место нового документа

При появлении нового текста ИИ автоматически определяет:

1. К какой системе относится (мир, созидатель, экосистема).
    
2. Какой тип мышления используется (смысл/архитектура/операции).
    
3. В какую ячейку поместить документ.
    
4. Нужно ли создавать новый документ или обновить существующий.
    
5. О каких связанных документах стоит упомянуть.
    

ИИ выводит рекомендацию в формате:

```
Рекомендуемая ячейка: 2.2 Архитектура созидателя.
Обоснование: текст описывает практики, режимы и структуру компетенций созидателя.
Действие: создать новый документ / обновить существующий.
Связанные документы: 2.1, 2.3, 3.2.
```

### 4.2 Проверять полноту хранилища

ИИ регулярно проводит аудит:

1. Какие ячейки заполнены?
    
2. Какие отсутствуют?
    
3. Где не хватает рабочих продуктов?
    
4. Где недостаточно связей?
    

ИИ формирует отчёт:

```
Отсутствуют документы:
— 1.3 Операции мира (не хватает описания внешних ритмов)
— 3.1 Смыслы экосистемы (нет анализа востребованности и роста токена)
Рекомендации по устранению:
— создать документ 1.3.1 ...
— обновить документ 3.1.1 ...
```

### 4.3 Проверять непротиворечивость

ИИ формирует автоматические отчёты:

— выявляет логические противоречия между документами;  
— определяет неопределённости или несогласованные формулировки;  
— проверяет соответствие архитектуры и операций смыслам.

Примеры:

```
Противоречие: 
— В документе 3.2.1 указано, что токены распределяются ежемесячно.
— В документе 3.3.2 указано, что распределение происходит ежеквартально.
Рекомендация: согласовать оба документа.
```

### 4.4 Рекомендовать обновления

ИИ анализирует:

— «устаревшие» разделы,  
— изменившиеся понятия,  
— нестыковки между компонентами системы.

ИИ формирует задания:

```
Документ 2.2.3 устарел: изменилась практика использования ИИ-агентов.
Рекомендуется обновить раздел «Интерфейсы созидателя».
```

## 5. Специальный раздел 0.4 — Автоматические отчёты ИИ

В разделе 0 должна существовать специальная папка:

```
0.4 Автоматические документы ИИ
```

В ней хранятся автоматически создаваемые ИИ файлы:

### 5.1 Отчёты о полноте хранилища

Список отсутствующих документов, слабых связей и незаполненных ячеек.

### 5.2 Отчёты о непротиворечивости

Формальные выявленные противоречия между документами.

### 5.3 Рекомендации по развитию хранилища

Приоритетные обновления, рекомендации по созданию новых документов и улучшению структуры.

### 5.4 Карта связей между документами

Граф зависимостей, формируемый ИИ.

### 5.5 Изменения во времени

Сводка обновлений, версий и эволюции содержания.

ИИ должен ежедневно формировать обновление раздела 0.4 или по запросу пользователя.

---

## 6. Поведение ИИ при работе с хранилищем

ИИ обязан:

1. Никогда не создавать документы вне структуры 0–3.
    
2. Всегда указывать предлагаемое место размещения.
    
3. Всегда указывать связанные документы.
    
4. Никогда не создавать текст, который ломает логику 3×3.
    
5. Обеспечивать непрерывную эволюцию хранилища без хаоса.
    

---

## 7. Итоговая схема работы хранилища

1. Уровень 0 — управление и автоматические отчёты.
    
2. Уровни 1–3 — строгая структура 3×3.
    
3. Каждый документ имеет своё место, свою роль, свои связи.
    
4. ИИ обеспечивает:  
    — классификацию,  
    — непротиворечивость,  
    — полноту,  
    — эволюцию.
