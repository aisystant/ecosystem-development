Этот документ объясняет правила структуры папок, именования, связей, версионирования и роли Obsidian↔Git; описывает «как здесь работают документы» и содержит полную таблицу классификации всех документов репозитория.

## Принципы организации хранения артефактов экосистемы

Репозиторий `ecosystem-development` является частью **гибридной архитектуры хранения артефактов** экосистемы. Полное описание архитектуры см. в [[0.3. Решения#ADR-003]].

### Что хранится в этом репозитории

**ecosystem-development** — это центральный репозиторий для **архитектурной документации высокого уровня** и **управленческих артефактов**. Здесь хранятся:

1. **Архитектурная документация (высокий уровень)**
   - ADR (Architecture Decision Records)
   - C4-диаграммы системного контекста
   - Описания подсистем на уровне черных ящиков
   - Интеграционная матрица подсистем
   - Общие принципы и паттерны

2. **Управленческие артефакты**
   - Входящие запросы (0.1)
   - План работ (0.2)
   - Решения / ADR (0.3)
   - Дорожная карта (0.4)
   - Проверочный документ (0.5)
   - Организация процессов (0.8)

3. **Идеи и концепция экосистемы**
   - Проблемы и гипотезы (1.1, 1.2)
   - Идеи и принципы развития (1.3)
   - Концепция функционирования (1.4)
   - Цели и задачи (1.5)

4. **Методология (высокий уровень)**
   - Картотека источников (3.1)
   - Программы развития (3.2)
   - Понятия (3.3)
   - Описание Memory Bank (3.4)

5. **Продвижение (высокий уровень)**
   - Целевые аудитории (2.1)
   - Принципы продвижения (2.3)
   - Манифест (2.6)

### Что НЕ хранится в этом репозитории

**Детали реализации подсистем** хранятся в отдельных репозиториях:

- **Код сервисов** → отдельные репозитории (agents-core, apps-sdk, platform, etc.)
- **Полные тексты руководств** → [aisystant/docs](https://github.com/aisystant/docs)
- **Операционные данные** → SurrealDB (метрики, логи, цифровые тени)
- **Контент для продвижения** → Google Drive (временно)
- **Управленческие задачи** → Plane

Подробная карта всех хранилищ экосистемы см. в [[0.9. Разное#Репозитории экосистемы]] (TODO: создать отдельный документ).

### Связь с другими репозиториями

```
ecosystem-development (этот репо)
    │
    ├── Ссылается на → aisystant/docs (полные тексты руководств)
    ├── Ссылается на → aisystant/platform (детали платформы интеграции)
    ├── Ссылается на → aisystant/agents-core (реализация агентов)
    ├── Ссылается на → aisystant/apps-sdk (SDK для ассистентов)
    ├── Ссылается на → aisystant/* (другие подсистемы)
    │
    └── Описывает → SurrealDB schema, ORY config, etc. (концептуально)
```

Принципы микросервисной архитектуры:
- **Автономия:** Каждый сервис сам выбирает где хранить свои операционные данные
- **Платформенные сервисы:** SurrealDB (цифровой след), ORY (аутентификация)
- **Документация:** Каждый репозиторий содержит README с описанием "прозрачного ящика"

## Как работать с классификацией

1. **Ручная правка в Obsidian**: измените значения в таблице напрямую, замените 🟡 на обычный текст
2. **Сохранение правок (WSL bash)**:
   ```bash
   python3 ops/save_manual_edits.py
   ```
3. **Повторная классификация (WSL bash)**:
   ```bash
   python3 ops/classify_documents.py
   ```
4. **Проверка валидности (WSL bash)**:
   ```bash
   python3 ops/validate_classifications.py
   ```

**Результат:** Отредактированные ячейки станут ЗЕЛЕНЫМИ (🟢) и ЗАЩИЩЕНЫ от изменений AI!

**Примечания:**
- Новые документы автоматически появятся при следующем запуске скрипта
- Изменения в документе 0.7 отразятся в таблице при обновлении
