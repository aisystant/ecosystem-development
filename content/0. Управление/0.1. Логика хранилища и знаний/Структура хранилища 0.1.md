# Структура хранилища 0.1

## 0. Назначение документа

Этот документ описывает полную структуру хранилища экосистемы, основанную на системной модели из трёх уровней и трёх ролей мышления.

---

## 1. Системная модель

Хранилище организовано на основе системной модели из трёх уровней:

```
┌─────────────────────────────────────────────────────────────────┐
│                     1. МИР (Надсистема)                         │
│         Контекст, в который встраивается созидатель             │
│              Проблемы мира, ЦА, партнёры, рынок                 │
└─────────────────────────────────────────────────────────────────┘
                              │
                              │ запрос на трансформацию
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                2. СОЗИДАТЕЛЬ (Целевая система)                  │
│              Человек, которого мы развиваем                     │
│         Модель, компетенции, путь, трансформация                │
└─────────────────────────────────────────────────────────────────┘
                              ▲
                              │ создаёт и развивает
                              │
┌─────────────────────────────────────────────────────────────────┐
│          3. ЭКОСИСТЕМА РАЗВИТИЯ (Система создания)              │
│     Платформа, команда, процессы, ИИ-ассистенты                 │
│         Инструменты для создания созидателей                    │
└─────────────────────────────────────────────────────────────────┘
```

---

## 2. Верхнеуровневая структура

| № | Раздел | Назначение |
|---|--------|------------|
| **0** | Управление | Метаинформация о хранилище (F0) |
| **1** | Мир (Надсистема) | Внешний контекст (F1, F2, F3) |
| **2** | Созидатель (Целевая система) | Кого создаём (F4, F5, F6) |
| **3** | Экосистема развития (Система создания) | Чем создаём (F7, F8, F9) |

---

## 3. Подразделы второго уровня

Для разделов 1-3 используется единая структура по трём ролям:

| Суффикс | Роль | Вопрос |
|---------|------|--------|
| X.1 | Предприниматель (Смыслы) | Зачем? Какая ценность? |
| X.2 | Инженер (Архитектура) | Как устроено? |
| X.3 | Менеджер (Операции) | Как работает/эксплуатируется? |

## 6. Принципы именования

### Разделы, подразделы, подподразделы

```
N. Название/                           # Раздел
N.M. Название — Пояснение/             # Подраздел
N.M.K. Название/                       # Подподраздел
```

### Документы

```
<Название документа> <N.M>.md
```

Примеры:
- `Манифест экосистемы 1.1.md`
- `Карта подсистем 3.2.md`
- `Централизованное хранилище 3.2.md`
