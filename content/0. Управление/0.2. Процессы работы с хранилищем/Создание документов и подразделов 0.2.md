# Создание документов и подразделов 0.2

## 0. Назначение документа

Этот документ описывает процесс создания новых документов и подразделов в хранилище. Он служит пошаговой инструкцией для людей и ИИ-агентов.

---

## 2. Принципы именования

### Разделы (верхний уровень)

```
N. Название/
```

Примеры:
- `0. Управление/`
- `1. Мир (Надсистема)/`
- `2. Созидатель (Целевая система)/`
- `3. Экосистема развития (Система создания)/`

### Подразделы (второй уровень)

```
N.M. Название — Пояснение/
```

Примеры:
- `1.1. Смыслы — Видение и привлечение/`
- `2.2. Архитектура — Модель и компетенции/`
- `3.3. Операции — Команда и службы/`

### Подподразделы (третий уровень)

```
N.M.K. Название/
```

Примеры:
- `1.1.1. Просвещение/`
- `3.2.3. ИТ-системы/`
- `3.3.2. Службы экосистемы/`

### Документы

```
<Название документа> <N.M>.md
```

**Важно:** Номер ячейки указывается **в конце** названия.

Примеры:
- `Манифест экосистемы 1.1.md`
- `Карта подсистем 3.2.md`
- `Централизованное хранилище 3.2.md`
- `Работа деканата 2.3.md`

### Информационные файлы

```
Информация N.M.md
```

Примеры:
- `Информация 0.0.md`
- `Информация 1.0.md`
- `Информация 2.0.md`

## 3. Процесс создания документа

### Шаг 1. Определить ячейку

Ответьте на два вопроса:

**Вопрос 1: К какой системе относится документ?**

| Вариант | Раздел | Примеры тем |
|---------|--------|-------------|
| Надсистема (Мир) | 1 | ЦА, рынок, партнёры, регуляторика |
| Целевая система (Созидатель) | 2 | Компетенции, маршрут, деканат |
| Система создания (Экосистема) | 3 | Платформа, команда, ИТ-системы |

**Вопрос 2: Какой тип мышления используется?**

| Вариант | Суффикс | Примеры тем |
|---------|---------|-------------|
| Смысловой (Зачем?) | .1 | Ценность, стратегия, бизнес-модель |
| Архитектурный (Как устроено?) | .2 | Модель, архитектура, интеграции |
| Операционный (Как работает?) | .3 | Процессы, команда, эксплуатация |

**Итог:** Ячейка = Раздел + Суффикс (например, 2.3 = Созидатель + Операции)

### Шаг 2. Выбрать название

- Название должно быть **содержательным** и отражать суть
- Избегайте абстрактных названий типа «Документ 1» или «Разное»
- Добавьте номер ячейки в конце: `<Название> <N.M>.md`

### Шаг 3. Создать файл из шаблона

Используйте шаблон из папки `0.3. Шаблоны/`:

```markdown
---
title: "Название документа"
family: F5                    # Семейство (F0-F9)
cell: "2.2"                   # Ячейка
status: draft                 # draft / working / approved
---

# Название документа N.M

## 1. Назначение

Кратко: для кого, зачем, какие задачи решает.

## 2. Содержание

Основная часть документа.

## 3. Связанные документы

- [[Документ 1]]
- [[Документ 2]]
```

### Шаг 4. Заполнить метаданные

Обязательные поля YAML-шапки:
- `title` — название документа
- `family` — семейство F0-F9
- `cell` — ячейка (например, "2.2")
- `status` — статус (draft / working / approved)

Опциональные поля:
- `depends_on` — родительские документы
- `affects` — зависимые документы
- `owner` — владелец

### Шаг 5. Добавить ссылки

В конце документа укажите связанные документы:
- Документы уровня выше (контекст)
- Документы уровня ниже (детализация)
- Соседние документы по строке/столбцу

## 4. Процесс создания подраздела

### Когда создавать подраздел

Создавайте подраздел (подпапку), если:
- Тема требует **3+ документов**
- Документы логически связаны
- Есть устойчивая потребность в группировке

### Шаги создания

1. **Определить место** — в какой ячейке будет подраздел
2. **Выбрать название** — `N.M.K. Название/`
3. **Создать папку** — `mkdir "N.M.K. Название"`
4. **Перенести документы** — если есть существующие
5. **Обновить структуру** — в документе [[Структура хранилища 0.1]]

### Примеры подразделов

```
1.1. Смыслы — Видение и привлечение/
├── 1.1.1. Просвещение/
├── 1.1.2. Маркетинг/
└── 1.1.3. Продажи/

3.2. Архитектура — Платформа и подсистемы/
├── 3.2.1. Концепция использования ИИ-платформы/
├── 3.2.2. Архитектурные решения/
├── 3.2.3. ИТ-системы/
└── 3.2.4. ИИ-ассистенты/
```

---

## 7. Типичные ошибки

| Ошибка | Как исправить |
|--------|---------------|
| Документ в неправильной ячейке | Определить систему и роль заново |
| Абстрактное название | Переименовать в содержательное |
| Нет номера ячейки в названии | Добавить `N.M` в конец |
| Нет YAML-шапки | Добавить из шаблона |
| Нет ссылок на связанные документы | Добавить раздел «Связанные документы» |
