# Структура проверочного документа 0.3.1

## 1. Назначение

Этот документ описывает структуру, по которой ИИ-ассистент собирает Проверочный документ (см. [[Проверочный документ 0.4]]).

---

## 2. Структура отчёта

Проверочный документ собирается по следующей структуре:

### Раздел 1. Зачем, для кого, что мы делаем и что меняется в мире
- Миссия экосистемы
- Целевая аудитория
- Ценностное предложение

### Раздел 2. Какие главные проблемы, гипотезы и цели мы видим
- Ключевые проблемы
- Рабочие гипотезы
- Стратегические цели

### Раздел 3. Как мы видим функционирование экосистемы
- Процессы и роли
- Взаимодействие участников

### Раздел 4. Структура ИИ-платформы
- Архитектура платформы
- Основные компоненты

### Раздел 5. Список подсистем
- Перечень всех подсистем (17+)
- Краткое описание каждой

### Раздел 6. Функциональная схема взаимодействия основных подсистем
- МИМ и контракты
- Потоки данных

### Раздел 7. Как организован процесс разработки
- 7-шаговый цикл работы
- Роли и ответственности

### Раздел 8. Как выглядит текущая классификация и какие теги у всех основных документов
- 4 оси классификации
- Используемые теги

### Раздел 9. Заключение о противоречиях
- Противоречия в идеях
- Несогласованность решений в разных документах
- Рекомендации по исправлению

---

## 3. Источники данных

Проверочный документ собирается из следующих исходных документов:

| Раздел отчёта | Источник |
|---------------|----------|
| 1. Миссия | Манифест экосистемы 1.1 |
| 2. Проблемы и цели | Гипотезы 1.1, Цели экосистемы 1.1 |
| 3. Функционирование | Концепция функционирования 1.1 |
| 4. Архитектура | Архитектура ИИ-платформы 3.2 |
| 5. Подсистемы | Карта подсистем 3.2, документы 3.2.3 |
| 6. Схема взаимодействия | Диаграммы интеграций 3.2 |
| 7. Процесс разработки | Организация процесса работы 0.2 |
| 8. Классификация | Классификация и теги 0.1 |
| 9. Противоречия | Анализ ИИ на основе всех документов |

---

## 4. Автоматизация

### Скрипт сборки
```bash
python3 ops/build_check_document.py
```

### GitHub Actions
- **Триггер при изменении:** `content/**/*.md`
- **Еженедельная сборка:** Понедельник, 9:00 UTC
- **Ручной запуск:** Через GitHub UI

### Результат
```
content/0. Управление/0.4. Автоматические отчёты ИИ/Проверочный документ 0.4.md
```
