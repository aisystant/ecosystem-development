### Слайд 1. ИИ — великий уравнитель

**Тезис:** Впервые в истории у каждого есть доступ к инструменту, который раньше был только у элит.

```
┌─────────────────────────────────────────────────────────────┐
│  Раньше (до 2023):                                          │
│  • Консультанты McKinsey — $1000/час                        │
│  • Личные ассистенты — для топ-менеджеров                   │
│  • Редакторы текстов — в издательствах                      │
│                                                             │
│  Сейчас:                                                    │
│  • Claude, GPT — доступны каждому                           │
│  • Цена: от бесплатно до $20-200/мес                        │
│  • Но... большинство использует на 5% возможностей         │
└─────────────────────────────────────────────────────────────┘
```

**Главная проблема:** ИИ даёт возможности, но без системы работы с ним — это просто чат.

---

### Слайд 2. Три составляющих созидателя

**Для созидателя критичны три вещи:**

| Составляющая | Что это | Без чего не работает |
|--------------|---------|---------------------|
| **Видение будущего** | Понимание, куда ты идёшь и зачем | Просто "найти работу" — слишком узко |
| **Агентность** | Способность действовать и менять реальность | Знания без действий = 0 |
| **Жизненное мастерство** | Мыслительное + профессиональное | Навыки без системы теряются |

**Ключевой вопрос:** Зачем вам это хранилище?
- Если просто найти работу — возможно, это overkill
- Если понять "за всё" — вы в правильном месте

### Слайд 8. FPF — операционная система для мышления ИИ

> **ДЕМОНСТРАЦИЯ 4:** Показать `CLAUDE.md` и `Claude Code и FPF 0.2.md`

**First Principles Framework (38K строк) — это:**
- Мета-теория рассуждений Анатолия Левенчука
- "Прошивка" для Claude Code
- Единый язык мышления для человека и ИИ

**Ключевые принципы:**

| Принцип | Описание |
|---------|----------|
| Holonic | Всё — холон (одновременно целое и часть) |
| Bounded Context | Значение локально в контексте |
| Strict Distinction | Роль ≠ Метод ≠ Работа; Документ ≠ Система |
| Trust Calculus | Доверие = ⟨Формальность, Область, Надёжность⟩ |
| ADI Cycle | Abduction → Deduction → Induction |

**Файлы:**
- `CLAUDE.md` — инструкции (~1K токенов, загружается всегда)
- `.fpf/FPF-Spec.md` — полная спецификация (~800K токенов, по частям)

   - Сверять терминологию

## ПРИЛОЖЕНИЕ А. ДЕТАЛЬНЫЕ СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ ХРАНИЛИЩА

### Сценарий А. "Ловля мысли" — от идеи до артефакта

**Контекст:** Вы на встрече, в душе, на прогулке — пришла важная мысль. Что делать?

**Проблема без системы:**
- Записал в заметки телефона — потерялось
- Отправил себе в Telegram — забылось
- Держал в голове — забылось через час

**Решение с хранилищем:**

```
┌─────────────────────────────────────────────────────────────────┐
│  ШАГ 1. ЗАХВАТ (0-30 секунд)                                    │
│  ─────────────────────────────                                  │
│  • Голосовое сообщение → транскрипция                           │
│  • Быстрая заметка в Obsidian Mobile                            │
│  • Фото доски/салфетки                                          │
│                                                                 │
│  Куда: content/0. Управление/0.9. Входящие/                     │
│  Формат: любой, без требований к структуре                      │
├─────────────────────────────────────────────────────────────────┤
│  ШАГ 2. КЛАССИФИКАЦИЯ (5-10 минут, позже)                       │
│  ─────────────────────────────────────────                      │
│  Вопросы к себе:                                                │
│  • Это про Мир, Созидателя или Экосистему?                      │
│  • Это Смыслы, Архитектура или Операции?                        │
│  • Кто целевая аудитория этой мысли?                            │
│                                                                 │
│  Результат: определена ячейка F1-F9                             │
├─────────────────────────────────────────────────────────────────┤
│  ШАГ 3. СТРУКТУРИРОВАНИЕ (15-30 минут)                          │
│  ──────────────────────────────────────                         │
│  • Создать файл в правильной папке                              │
│  • Добавить frontmatter из шаблона                              │
│  • Написать ядро документа (человек)                            │
│  • Развернуть детали (Claude Code)                              │
├─────────────────────────────────────────────────────────────────┤
│  ШАГ 4. ПУБЛИКАЦИЯ (5 минут)                                    │
│  ──────────────────────────                                     │
│  • git add → commit → push                                      │
│  • Создать PR                                                   │
│  • CI проверяет автоматически                                   │
│  • После ревью → merge                                          │
└─────────────────────────────────────────────────────────────────┘
```

**Пример диалога с Claude Code:**

```
Вы: "В 0.9 лежит заметка 'идея про геймификацию'.
     Помоги классифицировать и оформить."

Claude: Прочитал заметку. Это идея про мотивацию созидателей
        через игровые механики. Предлагаю:
        - Семейство: F6 (Путь и сопровождение)
        - Или F7 (Экономика) если про токены
        Какой аспект важнее?

Вы: "F6, это про путь"

Claude: Создаю документ в content/2.3. Операции/
        с frontmatter, структурой и ссылками на

### Сценарий Г. "Еженедельный обзор состояния"

**Контекст:** Каждую неделю нужно понимать: что изменилось, где пробелы, куда двигаться.

**Проблема без системы:**
- Статус-митинги на 2 часа
- "Кто что делал?" — каждый помнит своё
- Нет общей картины прогресса

**Решение с хранилищем:**

```
┌─────────────────────────────────────────────────────────────────┐
│  АВТОМАТИЧЕСКИЙ ЕЖЕНЕДЕЛЬНЫЙ ЦИКЛ                               │

│                                                                 │
│  ПОНЕДЕЛЬНИК (утро) — GitHub Actions запускает:                 │
│  ─────────────────────────────────────────────                  │
│  python3 ops/build_report.py --report all                       │
│                                                                 │
│  Генерируются отчёты:                                           │
│  • Архитектурный слепок — что есть                              │
│  • Содержательная полнота — где пробелы                         │
│  • Карта связей — как документы связаны                         │
│  • Терминология — согласованность терминов                      │
│  • Рекомендации — что делать дальше                             │
├─────────────────────────────────────────────────────────────────┤
│  ПОНЕДЕЛЬНИК (встреча команды) — 30 минут вместо 2 часов:       │
│  ─────────────────────────────────────────────────────────      │
│                                                                 │
│  1. Открыть "Архитектурный слепок" — тепловая карта             │
│     🟢🟢🟡🔴 — видно где хорошо, где плохо                       │
│                                                                 │
│  2. Посмотреть git log за неделю                                │
│     Кто, что, сколько коммитов                                  │
│                                                                 │
│  3. Открыть "Рекомендации" — топ-3 задачи на неделю             │

| Битых ссылок | Техническое качество |
| Терминологических расхождений | Согласованность |
