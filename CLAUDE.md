## Алгоритм создания любого документа

### ВАЖНО: Размещение новых документов

**Если пользователь НЕ указал конкретную папку для размещения документа, ВСЕГДА создавай документ в папке `content/0. Управление/0.9. Входящие/`.**

Папка «0.9. Входящие» — это inbox для черновиков и новых документов. После ревью документ будет перемещён в целевую ячейку.

### ПЕРЕД созданием документа Claude Code ОБЯЗАН:

1. **Загрузить управляющие документы** (см. таблицу ниже)
2. **Определить ячейку документа** (раздел + роль)
3. **Определить семейство F0-F9**
4. **Использовать шаблон и стандарты оформления**
5. **Применить ADI-цикл** для содержания (если нетривиальная задача)
6. **Разместить документ в `0.9. Входящие/`** (если пользователь не указал иное)

## Дополнительно: Архитектура мышления FPF

**FPF (First Principles Framework)** — методологическая основа для рассуждений и анализа.

### Когда загружать FPF

Загрузи соответствующую часть `.fpf/FPF-Spec.md` при:
- Анализе ролей, систем, архитектуры → Part A (строки 634-16099)
- Рассуждениях, оценке доверия → Part B (строки 16100-20246)
- Работе с терминологией → Part F (строки 32827-39500)
- Применении SoTA-методов → Part G (строки 39501-42007)

### ADI-цикл (для нетривиальных задач)

При создании содержания документа применяй цикл:
1. **Abduction** — сгенерируй 2-3 гипотезы
2. **Deduction** — проверь логику каждой
3. **Induction** — собери evidence из документов хранилища
4. **Audit** — проверь на когнитивные искажения
5. **Decide** — выбери лучшую гипотезу

### Навигация по FPF

- Индекс: `.fpf/INDEX.md`
- Спецификация: `.fpf/FPF-Spec.md` (~42K строк)

## Ключевые принципы (применяй всегда)

1. **Holonic (A.1):** Всё — холон (одновременно целое и часть)
2. **Bounded Context (A.1.1):** Значение локально в контексте
3. **Strict Distinction (A.7):** Роль ≠ Метод ≠ Работа; Документ ≠ Система
4. **Trust Calculus (B.3):** Доверие = ⟨F, G, R⟩
5. **ADI Cycle (B.5):** Abduction → Deduction → Induction
