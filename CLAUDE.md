### Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Holonic (A.1)** | Всё — холон (одновременно целое и часть) |
| **Bounded Context (A.1.1)** | Значение локально в контексте |
| **Strict Distinction (A.7)** | Роль ≠ Метод ≠ Работа; Документ ≠ Система |
| **Trust Calculus (B.3)** | Доверие = ⟨F, G, R⟩ |
| **ADI Cycle (B.5)** | Abduction → Deduction → Induction |

## Claude Code и FPF

### ADI-цикл для нетривиальных задач

При любой нетривиальной задаче **ОБЯЗАТЕЛЬНО** проходи цикл:

1. **Abduction** — сгенерируй 2-3 гипотезы (НЕ останавливайся на первой!)
2. **Deduction** — проверь логику каждой гипотезы
3. **Induction** — собери evidence из кода/документов
4. **Audit** — проверь на когнитивные искажения
5. **Decide** — выбери лучшую гипотезу, задокументируй rationale

### Применение FPF в коде

- При анализе архитектуры — загрузи Part A
- При проверке логики решений — применяй ADI-цикл
- При работе с терминологией — сверяйся с Part F
