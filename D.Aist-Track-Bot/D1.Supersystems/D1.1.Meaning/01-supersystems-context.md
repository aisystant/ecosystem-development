# Контекст надсистем Aist Track Bot

## Две надсистемы бота

Aist Track Bot функционирует в контексте **двух надсистем**, каждая из которых предъявляет свои требования и определяет свою ценность:

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Надсистема 1: ИТ-платформа Экосистемы (C2)                │
│  ├── Бот как компонент платформы                           │
│  └── Техническая интеграция, API, инфраструктура           │
│                                                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Надсистема 2: Киберличность Созидателя                    │
│  ├── Бот как часть персонального экзокортекса              │
│  └── Интеграция с Obsidian, заметками, знаниями            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
                    Aist Track Bot (D2)
```

---

## Надсистема 1: ИТ-платформа Экосистемы

### Что это

ИТ-платформа Экосистемы Aisystant (ядро C) — техническая инфраструктура, включающая веб-приложения, API, базы данных, интеграции.

### Роль бота в этой надсистеме

- **Точка входа** в экосистему через Telegram
- **Компонент платформы** с API-интеграциями
- **Канал доставки** образовательного контента

### Требования от ИТ-платформы

| Требование | Описание |
|------------|----------|
| Интеграция | API для синхронизации прогресса с основной платформой |
| Аутентификация | Связь Telegram-аккаунта с аккаунтом Aisystant |
| Метрики | Отправка данных в общую аналитику платформы |
| Масштабируемость | Поддержка роста пользователей |
| Надёжность | SLA, мониторинг, алерты |

### Ценность для платформы

- Расширение охвата (Telegram = 800M+ пользователей)
- Снижение порога входа (не нужна установка приложения)
- Удержание через push-уведомления

---

## Надсистема 2: Киберличность Созидателя

### Что это

**Киберличность** — цифровое расширение созидателя, включающее:
- Персональную базу знаний (Obsidian, Notion)
- Систему управления задачами
- Цифровой двойник (профиль компетенций)
- Инструменты мышления (экзокортекс)

### Роль бота в этой надсистеме

- **Проводник** в построении экзокортекса
- **Учитель** практик системного мышления
- **Валидатор** качества персональных артефактов

### Требования от Киберличности

| Требование | Описание |
|------------|----------|
| Интеграция с Obsidian | Проверка файлов из vault пользователя |
| Персонализация | Адаптация под уровень и цели созидателя |
| Приватность | Данные остаются у пользователя (local-first) |
| Долгосрочность | Spaced repetition, возврат к изученному |
| Трансферабельность | Экспорт прогресса и артефактов |

### Ценность для Киберличности

- Структурирование хаотичного обучения
- Формирование привычки системного мышления
- Создание работающей инфраструктуры знаний
- Связь изолированных заметок в граф

---

## Матрица "Надсистема × Режим бота"

| Режим | ИТ-платформа | Киберличность |
|-------|--------------|---------------|
| **Марафон** | Контент из платформы, синхронизация прогресса | Формирование начальных практик |
| **Лента** | Персонализация на основе данных платформы | Поддержание привычки чтения |
| **Экзокортекс** | Хранение метаданных артефактов | **Основной фокус**: сборка vault |

---

## Приоритизация надсистем по режимам

### Марафон и Лента

**Приоритет: ИТ-платформа**
- Контент поступает из платформы
- Прогресс синхронизируется с профилем
- Киберличность — вторичный контекст

### Экзокортекс

**Приоритет: Киберличность**
- Артефакты создаются локально (Obsidian)
- Платформа хранит только метаданные
- Фокус на построении персональной системы
