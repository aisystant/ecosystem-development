# Ecosystem Development

Хранилище знаний экосистемы развития: как мы помогаем людям становиться созидателями, способными менять мир к лучшему.

## Об экосистеме

**Экосистема развития** — это платформа, команда, процессы и ИИ-ассистенты, которые помогают людям трансформироваться в созидателей — людей с системным мышлением, способных видеть целое и создавать решения для сложных проблем.

**Главная миссия:** Сделать фундаментальное образование мирового уровня доступным для миллионов, сломав барьеры элитных университетов.

Это хранилище — **единый источник истины** о том, как устроена и работает экосистема.

## Системная модель

```
┌─────────────────────────────────────────────────────────────────┐
│                     1. МИР (Надсистема)                         │
│         Контекст, в который встраивается созидатель             │
│              Проблемы мира, ЦА, партнёры, рынок                 │
└─────────────────────────────────────────────────────────────────┘
                       │                 ▲
       запрашивает     │                 │  изменяет мир,
       трансформацию   │                 │  создаёт ценность
                       ▼                 │
┌─────────────────────────────────────────────────────────────────┐
│                2. СОЗИДАТЕЛЬ (Целевая система)                  │
│              Человек, которого мы развиваем                     │
│         Модель, компетенции, путь, трансформация                │
└─────────────────────────────────────────────────────────────────┘
                       ▲                 │
       создаёт и       │                 │  развивает сообщество,
       развивает       │                 │  даёт обратную связь
                       │                 ▼
┌─────────────────────────────────────────────────────────────────┐
│          3. ЭКОСИСТЕМА РАЗВИТИЯ (Система создания)              │
│     Платформа, команда, процессы, ИИ-ассистенты                 │
│         Инструменты для создания созидателей                    │
└─────────────────────────────────────────────────────────────────┘
```

**Три уровня системы × Три роли мышления** — это **SRT-метод** (Systems-Roles-Table), основанный на системном мышлении:

|              | Смыслы (зачем?) | Архитектура (как устроено?) | Операции (как работает?) |
|--------------|-----------------|----------------------------|--------------------------|
| **1. Мир**   | 1.1 Продвижение | 1.2 Контексты применения созидателя | 1.3 Репутация и партнёрства |
| **2. Созидатель** | 2.1 Ценность | 2.2 Модель и компетенции | 2.3 Путь и сопровождение |
| **3. Экосистема** | 3.1 Экономика | 3.2 Платформа и подсистемы | 3.3 Команда и службы |

## Что даёт экосистема

| Что получает | Как это работает |
|--------------|------------------|
| **Персональный маршрут развития** | ИИ-Проводник анализирует цели и подбирает программу |
| **Практические компетенции** | Руководства + стажировки + резидентуры |
| **Цифровой двойник** | Отслеживание прогресса, компетенций, достижений |
| **Сообщество единомышленников** | Клуб, менторы, совместные проекты |
| **Токены за вклад** | Вознаграждение за подтверждённый вклад в развитие |

### Команде экосистемы

| Что получает | Как это работает |
|--------------|------------------|
| **Единый источник истины** | 232 документа структурированы по системной модели |
| **Автоматические отчёты ИИ** | Архитектурный слепок, полнота, противоречия, связи |
| **Контекст для ИИ-ассистентов** | Хранилище — Memory Bank для всех агентов |

## Три слоя экосистемы

**1. Платформа (технологии)**
- 14+ детерминированных ИТ-систем (LMS, CRM, Биллинг, Цифровой двойник и др.)
- 17 ИИ-агентов для автоматизации процессов (включая Индексатор знаний)
- 6 ИИ-ассистентов для персонального сопровождения (Проводник и др.)
- Хранилища данных: [docs](https://github.com/aisystant/docs), [personal-knowledge](https://github.com/aisystant/personal-knowledge), MCP-серверы ([fsm-mcp](https://github.com/aisystant/fsm-mcp), [digital-twin-mcp](https://github.com/aisystant/digital-twin-mcp))

**2. Команда (люди)**
- Службы: Школа, Лаборатории, Мастерская, Клуб, Деканат, Продюсерский центр
- Роли: методологи, кураторы, менторы, разработчики

**3. Экономика (вознаграждение)**
- Токены за подтверждённый вклад
- Репутация через epistemic status
- Доступ к ресурсам по вкладу

## Принципы работы

1. **Единое место для документа** — каждый документ имеет одну правильную ячейку
2. **Человек создаёт и редактирует ядро** — цели, видение, основные решения
3. **ИИ генерирует производные** — отчёты, индексы, проверки, предложения
4. **Все изменения через PR** — для трассируемости и ревью
5. **Регулярный рефакторинг** — хранилище как живая система

## Архитектура мышления: First Principles Framework (FPF)

Экосистема использует **First Principles Framework** — мета-теорию для рассуждений, разработанную Анатолием Левенчуком. FPF работает как «операционная система для мышления» (~42K строк спецификации).

### Структура FPF

| Часть | Описание | Применение |
|-------|----------|------------|
| **Part A** | Kernel — Holon, Role, Method, Work | Анализ ролей и систем |
| **Part B** | Reasoning — Trust Calculus, ADI-цикл | Рассуждения и доверие |
| **Part C** | Architheories — формальные спецификации | Моделирование доменов |
| **Part F** | Unification — терминология, мосты | Унификация контекстов |
| **Part G** | SoTA Kit — современные методы | Работа с практиками |

### Ключевые принципы

| Принцип | Описание |
|---------|----------|
| **Holonic (A.1)** | Всё — холон (одновременно целое и часть) |
| **Bounded Context (A.1.1)** | Значение локально в контексте |
| **Strict Distinction (A.7)** | Роль ≠ Метод ≠ Работа; Документ ≠ Система |
| **Trust Calculus (B.3)** | Доверие = ⟨F, G, R⟩ (формальность, область, надёжность) |
| **ADI Cycle (B.5)** | Abduction → Deduction → Induction |

### Цикл рассуждения (ADI)

1. **Abduction** — генерируй 2-3 гипотезы (не якорись на первой!)
2. **Deduction** — проверь логику и ограничения
3. **Induction** — собери evidence
4. **Audit** — проверь на bias
5. **Decide** — выбери и задокументируй rationale

Навигация: [`.fpf/INDEX.md`](.fpf/INDEX.md) • Спецификация: [`.fpf/FPF-Spec.md`](.fpf/FPF-Spec.md)

## Claude Code и FPF

Репозиторий интегрирован с **Claude Code** для работы с First Principles Framework.

### Архитектура контекста

```
CLAUDE.md (~1K токенов)              ← загружается автоматически
    ├── Ключевые принципы FPF
    ├── Терминология проекта
    └── Таблица навигации по FPF

.fpf/INDEX.md                        ← индекс для навигации
    └── Карта частей FPF-Spec.md

.fpf/FPF-Spec.md (~42K строк)        ← загружается по частям
    └── Полная спецификация FPF
```

### Как это работает

### Интеграция с ИИ-платформой

Экосистема использует **MCP-серверы** (Model Context Protocol) для доступа ИИ-ассистентов к данным.

**Архитектура интеграции:**

```
Пользовательский интерфейс (UI — "интерфейс")
(OpenAI Apps SDK / LMS / Telegram)
         ↓
AI Runtime / Orchestrator (исполнитель/оркестратор ИИ)
(агенты и ассистенты, контекст, генерация, QA)
         ↓
Tool Layer via MCP (слой инструментов через MCP)
(MCP-серверы: поиск, чтение, запись, внешние API)
         ↓
Data Layer (слой данных)
(БД, репозитории, векторные индексы, файлы)
```

**MCP-серверы (Tool Layer):**

| MCP-сервер           | Назначение                                                                                                      |
| -------------------- | --------------------------------------------------------------------------------------------------------------- |
| **fsm-mcp**          | Логика работы Ассистента ученика — FSM-инструкции и переходы ([GitHub](https://github.com/aisystant/fsm-mcp))   |
| **digital-twin-mcp** | Доступ к данным цифрового двойника созидателя ([GitHub](https://github.com/aisystant/digital-twin-mcp))         |
| **guides-mcp**       | Доступ к репозиторию руководств для ДЗ-чекера и других агентов                                                  |

**Пример реализации:** [Ассистент ученика](content/3.%20Экосистема%20развития%20(Система%20создания)/3.2.%20Архитектура%20—%20Платформа%20и%20подсистемы/3.2.4.%20ИИ-ассистенты/3.2.4.1.%20Проводник%20по%20персональному%20маршруту%20(Ассистент%20ученика)/README.md) построен на основе OpenAI Apps SDK + fsm-mcp.

Подробнее: [fsm-mcp README](https://github.com/aisystant/fsm-mcp/blob/main/README.md)
