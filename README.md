# Ecosystem Development

Хранилище знаний экосистемы развития: как мы помогаем людям становиться созидателями — профессионалами, способными менять мир к лучшему.

## Два типа созидателей

### Что даёт экосистема созидателю (пользователю)?

**Созидатель** — это человек, который осознанно развивает себя и создаёт системы, меняющие мир. Экосистема даёт ему:

| Что получает | Как это работает |
|--------------|------------------|
| **Персональный маршрут развития** | ИИ-Проводник анализирует цели и подбирает программу |
| **Практические компетенции** | Руководства + стажировки + резидентуры |
| **Цифровой двойник** | Отслеживание прогресса, компетенций, достижений |
| **Сообщество единомышленников** | Клуб, менторы, совместные проекты |
| **Инструменты для работы** | ИИ-ассистенты, шаблоны, методики |

### Что даёт это хранилище созидателю экосистемы (команде)?

**Созидатель экосистемы** — это член команды, который строит и развивает саму экосистему. Хранилище даёт ему:

| Что получает | Как это работает |
|--------------|------------------|
| **Единый источник истины** | Все знания о проекте в одном месте, структурированы по системной модели |
| **Общий язык команды** | Глоссарий, словарь сущностей, единая терминология |
| **Архитектурные решения** | ADR с контекстом, альтернативами и последствиями |
| **Автоматические отчёты** | ИИ проверяет полноту, противоречия, связи между документами |
| **Онбординг новых участников** | Структура 0-1-2-3 позволяет быстро понять проект |
| **Трассируемость изменений** | Git-история, PR-ревью, версионирование |
| **Контекст для ИИ-ассистентов** | Хранилище — это Memory Bank для всех агентов |

**Зачем это нужно?** Без единого хранилища знания рассыпаны по головам, чатам и документам. Хранилище делает неявное явным и позволяет команде работать асинхронно, а ИИ — помогать эффективно.

                       ▲                 │

## О проекте

Это единое хранилище знаний (Memory Bank) для разработки экосистемы развития. Репозиторий содержит:

- **Философию и методологию** — зачем существует экосистема и как она работает
- **Архитектуру платформы** — около 20 ИТ-систем и ИИ-ассистентов
- **Программы развития** — руководства, стажировки, резидентуры
- **Управленческие артефакты** — процессы, планы, отчёты

## Быстрый старт

### Работа в Obsidian

1. Клонируйте репозиторий
2. Откройте папку `content/` как vault в Obsidian
3. Установите плагины:
   - **Obsidian Git** — синхронизация с GitHub
   - **Dataview** — динамические запросы
4. Используйте wikilinks для связей между документами

### Внесение изменений

1. Создайте feature-ветку
2. Внесите изменения (новый документ → в `0.9. Входящие/`)
3. Откройте Pull Request
4. После ревью — merge в main

Подробнее: [Процесс внесения изменений 0.2](content/0.%20Управление/0.2.%20Процессы%20работы%20с%20хранилищем/Процесс%20внесения%20изменений%200.2.md)

## Принципы работы

1. **Единое место для документа** — каждый документ имеет одну правильную ячейку
2. **Человек редактирует ядро** — цели, видение, решения
3. **ИИ генерирует производные** — отчёты, индексы, проверки
4. **Все изменения через PR** — для трассируемости и ревью
5. **Регулярный рефакторинг** — хранилище как живая система

## Автоматизация

```bash
# Генерация всех отчётов
python3 ops/build_report.py --report all

# Отдельные отчёты
python3 ops/build_report.py --report architecture-snapshot
python3 ops/build_report.py --report content-completeness
python3 ops/build_report.py --report technical-issues
python3 ops/build_report.py --report links-map
```

GitHub Actions автоматически:
- Валидирует frontmatter метаданных
- Проверяет markdown линтером
- Генерирует отчёты по расписанию
