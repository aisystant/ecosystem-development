# Ecosystem Development

Хранилище знаний экосистемы развития: как мы помогаем людям становиться созидателями, способными менять мир к лучшему.

## Об экосистеме

**Экосистема развития** — это платформа, команда, процессы и ИИ-ассистенты, которые помогают людям трансформироваться в созидателей — людей с системным мышлением, способных видеть целое и создавать новые решения.

**Главная миссия:** Сделать фундаментальное образование мирового уровня доступным для миллионов, сломав барьеры элитных университетов.

Это хранилище — **единый источник истины** о том, как устроена и работает экосистема.

## Что даёт экосистема

### Созидателю (пользователю)

| Что получает | Как это работает |
|--------------|------------------|
| **Персональный маршрут развития** | ИИ-Проводник анализирует цели и подбирает программу |
| **Практические компетенции** | Руководства + стажировки + резидентуры |
| **Цифровой двойник** | Отслеживание прогресса, компетенций, достижений |
| **Сообщество единомышленников** | Клуб, менторы, совместные проекты |
| **Токены за вклад** | Вознаграждение за подтверждённый вклад в развитие |

### Команде экосистемы

| Что получает | Как это работает |
|--------------|------------------|
| **Единый источник истины** | Все знания о проекте структурированы по системной модели |
| **Автоматические отчёты** | ИИ проверяет полноту, противоречия, связи |
| **Контекст для ИИ-ассистентов** | Хранилище — Memory Bank для всех агентов |

## Три слоя экосистемы

**1. Платформа (технологии)**
- 14+ детерминированных ИТ-систем (LMS, CRM, Биллинг, Цифровой двойник и др.)
- 17 ИИ-агентов для автоматизации процессов
- 6 ИИ-ассистентов для персонального сопровождения (Проводник и др.)

**2. Команда (люди)**
- Службы: Школа, Лаборатории, Мастерская, Клуб, Деканат
- Роли: методологи, кураторы, менторы, разработчики

**3. Экономика (вознаграждение)**
- Токены за подтверждённый вклад
- Репутация через epistemic status
- Доступ к ресурсам по вкладу

## Быстрый старт

### Работа в Obsidian или другом редакторе md

1. Клонируйте репозиторий
2. Откройте папку `content/` как vault в Obsidian (или откройте в любом markdown-редакторе)
3. Для Obsidian установите плагины:
   - **Obsidian Git** — синхронизация с GitHub
   - **Dataview** — динамические запросы
4. Используйте wikilinks для связей между документами

### Внесение изменений

1. Создайте feature-ветку
2. Внесите изменения (новый документ → в `0.9. Входящие/`)
3. Откройте Pull Request
4. После ревью — merge в main

## Принципы работы

1. **Единое место для документа** — каждый документ имеет одну правильную ячейку
2. **Человек создаёт и редактирует ядро** — цели, видение, основные решения
3. **ИИ генерирует производные** — отчёты, индексы, проверки, предложения
4. **Все изменения через PR** — для трассируемости и ревью
5. **Регулярный рефакторинг** — хранилище как живая система

## Claude Code и FPF

Репозиторий интегрирован с **Claude Code** и **First Principles Framework** (FPF) Анатолия Левенчука.

### Архитектура контекста

```
CLAUDE.md (~1K токенов)              ← загружается автоматически
    ├── Ключевые принципы FPF
    ├── Терминология проекта
    └── Таблица навигации по FPF

.fpf/INDEX.md                        ← индекс для навигации
    └── Карта частей FPF-Spec.md

.fpf/FPF-Spec.md (~800K токенов)     ← загружается по частям
    └── Полная спецификация FPF
```

### Как это работает

1. **CLAUDE.md** — инструкции для Claude Code с ключевыми принципами FPF
2. **INDEX.md** — навигационный индекс для выборочной загрузки частей FPF
3. **FPF-Spec.md** — полная спецификация (читается по частям через offset/limit)

### Обновление FPF

```bash
# Mac / Linux
curl -fsSL https://raw.githubusercontent.com/ailev/FPF/main/FPF-Spec.md -o .fpf/FPF-Spec.md
git add .fpf && git commit -m "chore: update FPF" && git push
```
