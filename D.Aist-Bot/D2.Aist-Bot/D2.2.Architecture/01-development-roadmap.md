## Roadmap: Режим Экзокортекс (v2.0)

### Концепция

**Экзокортекс** — третий режим бота, отличающийся от Марафона и Ленты:

| Режим | Прогресс измеряется | Метафора |
|-------|---------------------|----------|
| Марафон | По времени (дни) | Спринт |
| Лента | По потоку (темы) | Река |
| **Экзокортекс** | По артефактам (РП) | Конструктор |

**Ключевой дизайн:** Каждое задание заканчивается **рабочим продуктом (РП)**, который можно проверить автоматически.

### Duolingo-подобная модель прогресса

```
Path (Путь)
  └── Level (Уровень)
        └── Step (Шаг)
              └── Check (Проверка)
                    └── Mastery (Мастерство)
```

| Единица | Описание | Пример |
|---------|----------|--------|
| **Path** | Набор уровней | "Экзокортекс v1" |
| **Level** | 5-12 шагов | "L01 Foundation", "L02 Capture" |
| **Step** | 1 рабочий продукт | "Создать inbox заметок" |
| **Check** | Автопроверка/рубрика | "Проверка структуры frontmatter" |
| **Mastery** | Spaced repetition | Повторная проверка через интервал |

### Состояние пользователя

```yaml
user_state:
  current_path: "exocortex_v1"
  current_level: "L02"
  current_step: "S03"
  steps:
    - step_id: "L01_S01"
      status: "passed"
      artifact_link: "user_vault/inbox.md"
    - step_id: "L01_S02"
      status: "passed"
    - step_id: "L02_S01"
      status: "in_progress"
  streak: 5
  review_queue: ["L01_S03", "L01_S05"]
```

## Архитектура Экзокортекса

### Obsidian-First подход

**Почему Obsidian:**
- Артефакты = Markdown файлы (легко проверять)
- Starter vault как шаблон
- Git-вариант для продвинутых
- Локальное хранение = приватность

**MVP для широкой аудитории:**

```
1. Пользователь скачивает starter vault (zip)
2. Бот выдаёт шаг → описание РП
3. Пользователь:
   - Создаёт файл по шаблону
   - Присылает файл / текст / ссылку на gist
4. Бот прогоняет автопроверки
5. "Пройдено" или "Что исправить" + подсказки
```

### Логические артефакты (платформо-независимые)

Не привязываемся к конкретной платформе:

| Логический РП | Obsidian-адаптер | Notion-адаптер (позже) |
|---------------|------------------|------------------------|
| `episteme_note` | MD + frontmatter | Page + DB properties |
| `role_table` | MD таблица | Database |
| `inbox` | Папка + шаблон | Database с фильтром |
| `planning_structure` | Daily notes | Timeline database |

## Структура курса Экзокортекса

### Три ступени сложности

#### Ступень 1: Экзокортекс Lite (2-4 часа)

**Цель:** Работающий цикл "чтение → capture → заметка → поиск"

**Рабочие продукты:**
- Базовая таблица ролей/держателей
- Inbox заметок
- 5-10 "эпистем" (заметок) с минимальными полями

**Шаги:**
1. S01: Установка Obsidian + starter vault
2. S02: Понимание структуры vault
3. S03: Первая заметка по шаблону
4. S04: Таблица "Роль × Держатель"
5. S05: Inbox и capture-процесс
6. S06: 5 эпистем из одного источника
7. S07: Поиск и связывание заметок

#### Ступень 2: Экзокортекс Standard (2-4 недели)

**Цель:** Полный цикл "эпистема → план → действие → ретро → обновление"

**Рабочие продукты:**
- Корпус эпистем (каталог)
- Планировочная структура
- Протоколы ретро и обновления

**Уровни:**
- L02: Capture (захват информации)
- L03: Process (обработка в эпистемы)
- L04: Plan (планирование на основе знаний)
- L05: Review (ретроспективы)
- L06: Iterate (обновление корпуса)

#### Ступень 3: Экзокортекс Pro (для глубокого FPF)

**Цель:** Формальная типизация, контекстная дисциплина

**Рабочие продукты:**
- Формальные шаблоны `holder#role:context`
- Таксономии эпистем (definition/evidence/standard/claim)
- Метрики качества экзокортекса

## Автопроверки, обучающие FPF

### Типы проверок

| Тип | Описание | Пример |
|-----|----------|--------|
| **Структурные** | Формат, поля, связи | "Frontmatter содержит все обязательные поля" |
| **Онтологические** | FPF-ограничения | "Если держатель = Артефакт, роль не деятельностная" |
| **Педагогические** | Объяснение ошибки | "Почему это неверно + пример исправления" |

### Примеры FPF-проверок

```yaml
checks:
  - name: "holder_role_consistency"
    rule: |
      Если Тип держателя = Артефакт,
      то роль НЕ может быть деятельностной (читать/собирать/планировать)
    error_message: |
      Артефакт не может выполнять действия.
      Пример: "Заметка" — это артефакт, она не "читает".
      Исправьте: укажите роль как "Носитель эпистемы".

  - name: "system_vs_process"
    rule: |
      "Жизнь" не может быть названа системой в конкретном контексте
    error_message: |
      Не путайте процесс и систему.
      "Жизнь" — это процесс или контекст, не система.
```

## Фазы реализации

### Фаза 1: MVP Экзокортекс Lite (4-6 недель)

**Задачи:**
- [ ] Спроектировать starter vault Obsidian
- [ ] Определить первые 7 шагов Lite
- [ ] Реализовать state machine прогресса
- [ ] Базовые структурные проверки
- [ ] Интеграция с существующим ботом

**Критерии готовности:**
- Пользователь может пройти L01 от начала до конца
- Прогресс сохраняется между сессиями
- Минимум 3 автопроверки работают

### Фаза 2: Полный Экзокортекс Standard (8-12 недель)

**Задачи:**
- [ ] Уровни L02-L06
- [ ] FPF-проверки (онтологические)
- [ ] Педагогические подсказки
- [ ] Spaced repetition (Mastery)
- [ ] Метрики и аналитика

### Фаза 3: Pro + Notion (12+ недель)

**Задачи:**
- [ ] Формальные шаблоны Pro
- [ ] Notion-адаптер
- [ ] API для внешних интеграций
- [ ] Наставнический дашборд
