# Мастер-онтология экосистемы (общая)

> **Тип:** Агрегированный индекс всех общих понятий экосистемы
> **Назначение:** Единая точка навигации по онтологиям общих Pack. Каждое понятие ссылается на Pack, где оно определено.
> **Персональная онтология (PD):** Не входит сюда. У каждого пользователя свой `DS-my-strategy/ontology.md` со ссылкой на этот файл.
> **Синхронизация:** Процесс `ontology-sync` (DS-ai-systems/extractor/prompts/ontology-sync.md)
>
> **Два пути доступа:**
> - **Новый пользователь** (из шаблона экзокортекса): получает SPF/ontology.md → создаёт свою PD-онтологию в DS-my-strategy/
> - **Член команды**: получает доступ к этому файлу (DS-ecosystem-development/) → видит все общие понятия (DP+ECO+MIM)
>
> **Правило двунаправленности:**
> - Новое понятие в Pack → должно появиться здесь (со ссылкой)
> - Понятие здесь → должно ссылаться на Pack, где определено

---

## 1. Архитектура онтологий (4 уровня)

```
FPF (мета-онтология, U.*)
  └─ SPF (правила формы, SPF.SPEC.*)
       └─ Pack (знания домена, КОНТЕКСТ.ВИД.*)
            └─ Downstream (ссылки на Pack, без новых понятий)
```

| Уровень | Файл | Что определяет |
|---------|------|---------------|
| FPF | [FPF](https://github.com/ailev/FPF) | U.* — универсальные понятия |
| SPF | [SPF/ontology.md](../SPF/ontology.md) | Правила структуры онтологий (SPEC.002) |
| Pack: DP | [PACK-digital-platform/ontology.md](../PACK-digital-platform/ontology.md) | ИТ-платформа: архитектура, агенты, навигация |
| Pack: ECO | [PACK-ecosystem/ontology.md](../PACK-ecosystem/ontology.md) | Экосистема: компоненты, бесшовность, переходы |
| Pack: MIM | [PACK-MIM/ontology.md](../PACK-MIM/ontology.md) | Мастерская: форматы, программы, организация |
| Pack: EDU | [PACK-education/ontology.md](../PACK-education/ontology.md) | Образование: методы обучения, оценивание, адаптация |

## 3. Кросс-Pack связи

| Понятие A | Pack A | Понятие B | Pack B | Связь |
|-----------|--------|-----------|--------|-------|
| ИИ-система (агент) | DP | Слайдомент | MIM | AI-агент работает с материалом семинара |
| Платформа | DP | Бесшовность | ECO | Платформа обеспечивает техническую бесшовность |
| Экстракция знаний | DP | Знание | PD* | Экстрактор формализует знания в Pack |
| Программы развития | MIM | Компоненты экосистемы | ECO | Программы МИМ — подсистема экосистемы |
| Среда (IWE) | DP | Экосистема | ECO | Личные среды — компоненты экосистемы, связанные с Платформой |
| Среда (IWE) | DP | Платформа | DP | Среда использует модули Платформы + собственные модули пользователя |
| Методы обучения | EDU | Форматы мероприятий | MIM | EDU-методы применяются внутри MIM-форматов |
| Backwards Design | EDU | Deliberate Practice | MIM | Backwards Design проектирует обучение, DP — тренировку |
| Scaffolding | EDU | Роль Просветитель | PD* | Просветитель использует методы из EDU |
| Retrieval Practice | EDU | Слайдомент | MIM | Retrieval practice встраивается в семинар для проверки усвоения |
| Spaced Repetition | EDU | Программа развития | MIM | Программы MIM должны учитывать spacing при планировании занятий |

> *PD (Personal Development) — персональный Pack. Не входит в эту общую онтологию, хранится в личном `DS-my-strategy/ontology.md`.
