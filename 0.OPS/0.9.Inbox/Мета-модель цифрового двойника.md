# Мета-модель цифрового двойника

> **Версия:** 0.1 (начальная концепция)

## 0. Назначение, аудитория, границы

**Назначение документа:** описать мета-модель цифрового двойника стажёра — структуру данных, типы показателей и MCP-инструменты для работы с ними.

**Аудитория:**
- Разработчики платформы и MCP-серверов
- Методологи, проектирующие модель созидателя
- ИИ-агенты, работающие с данными стажёра

**Границы:** документ описывает структуру мета-модели (схему данных), а не конкретные значения. Фактические данные стажёров хранятся в отдельном репозитории цифрового двойника.

**Связь с репозиторием ecosystem-development:**
- Мета-модель относится к **семейству F5** (Architecture × SoI) — модель созидателя
- Коды показателей (i.*) согласованы с существующей нотацией репозитория
- Интеграция с Проводником по персональному маршруту через профиль стажёра

## 1. Концепция мета-модели

### 1.1. Что такое мета-модель?

**Мета-модель** — это описание структуры данных цифрового двойника:
- Какие показатели существуют
- Какие у них типы и форматы
- Как они организованы иерархически
- Когда и зачем они используются

**Цифровой двойник** — это набор данных о конкретном стажёре, соответствующий мета-модели.

```
Мета-модель (schema)     →    Цифровой двойник (data)
─────────────────────────────────────────────────────
i.role_set: List[str]    →    ["писатель", "читатель"]
i.daily_task_time: Time  →    "08:00"
i.qualification_level    →    "Систематический"
```

### 1.2. Принцип организации: иерархия i/

Все показатели цифрового двойника организованы в иерархическую структуру с корнем `i/` (от "individual"):

```
i/
├── i.objective              # Цели обучения
├── i.role_set               # Текущие роли
├── i.domain_focus           # Интересующие области
├── i.daily_task_time        # Время получения задания
├── i.daily_report_time      # Время отчёта за день
├── plan/                    # План развития
│   ├── i.target_state
│   ├── i.guide_track_set
│   └── ...
├── qualification/           # Квалификация
│   ├── i.qualification_level
│   └── ...
├── agency/                  # Агентность
│   ├── i.development_regularity
│   └── ...
└── ...
```

### 1.3. Типы показателей

| Тип показателя | Описание | Примеры кодов |
|----------------|----------|---------------|
| **Семантический** | Текстовые описания, списки строк | i.objective, i.role_set |
| **Структурный** | Форматированные значения (время, дата) | i.daily_task_time, i.weekly_analytics_time |
| **Категориальный** | Выбор из предопределённых значений | i.qualification_level |
| **Частотный** | Метрики частоты действий | i.development_regularity |
| **Шкальный** | Числовые оценки по шкале | i.target_state (1-5) |
| **Временной** | Длительности, даты | i.time_invested |

### 2.2. Формат описания показателя

Каждый показатель описывается в отдельном файле:

```markdown
# i.role_set

## Мета-данные

| Свойство | Значение |
|----------|----------|
| **Код** | i.role_set |
| **Название** | Текущие роли |
| **Тип** | Семантический |
| **Формат** | List[str] — список строк |
| **Группа** | Общие данные |
| **Для промптов** | Да |
| **Модель стажера** | Да |

## Описание

Список текущих ролей, которые стажёр выполняет в жизни и работе.
Используется для персонализации заданий и примеров.

## Примеры значений

```json
["писатель", "читатель", "слушатель"]
["менеджер проекта", "отец", "студент"]
["разработчик", "тимлид", "наставник"]
```

## Использование

- **Проводник:** адаптация примеров под роли стажёра
- **Генератор контента:** выбор релевантных кейсов
- **Аналитика:** сегментация стажёров по ролевому профилю

## Связи

- Влияет на: i.domain_focus, i.highlighted_methods
- Зависит от: —
```

## 3. MCP-инструменты

### 3.1. Инструменты мета-модели

Для работы со структурой мета-модели (схемой):

#### describe_by_path(path)

Возвращает описание показателей по указанному пути.

**Параметры:**
- `path` (string): путь в иерархии, например `"i"`, `"i/agency"`, `"i.role_set"`

**Примеры:**

```
describe_by_path('i')
```
Результат:
```
objective: semantic: Цели обучения
role_set: semantic: Текущие роли
domain_focus: semantic: Интересующие области
daily_task_time: structural: Время получения задания
plan/: group: План развития (6 показателей)
qualification/: group: Квалификация (4 показателя)
agency/: group: Агентность (5 показателей)
mastery/: group: Степень мастерства (8 показателей)
resourcefulness/: group: Поддерживающая ресурсность (5 показателей)
adaptability/: group: Адаптивность и устойчивость (4 показателя)
history/: group: История (4 показателя)
```

```
describe_by_path('i/agency')
```
Результат:
```
development_regularity: frequency: Регулярность саморазвития
execution_stability: frequency: Устойчивость слотов саморазвития
reflection_frequency: frequency: Частота рефлексий
strategy_regularity: frequency: Регулярность стратегирования
systemic_levels_of_intent: semantic: Системные уровни неудовлетворённостей
```

```
describe_by_path('i.role_set')
```
Результат:
```
code: i.role_set
name: Текущие роли
type: semantic
format: List[str]
description: Список текущих ролей стажёра
for_prompts: true
example: ["писатель", "читатель", "слушатель"]
```

#### list_indicators(filter)

Возвращает список показателей по фильтру.

**Параметры:**
- `filter` (object): критерии фильтрации
  - `group` (string, optional): группа показателей
  - `type` (string, optional): тип показателя
  - `for_prompts` (boolean, optional): только для промптов

**Пример:**
```
list_indicators({group: "agency", for_prompts: true})
```
Результат:
```json
[
  {"code": "i.development_regularity", "name": "Регулярность саморазвития"},
  {"code": "i.execution_stability", "name": "Устойчивость слотов"}
]
```

### 3.2. Инструменты цифрового двойника

Для работы с данными конкретного стажёра:

#### read_digital_twin(path)

Читает значение показателя из цифрового двойника.

**Параметры:**
- `path` (string): путь к показателю, например `"i.role_set"`, `"i.agency.development_regularity"`

**Примеры:**

```
read_digital_twin('i.role_set')
```
Результат:
```json
["писатель", "читатель", "слушатель"]
```

```
read_digital_twin('i.qualification_level')
```
Результат:
```json
"Систематический"
```

```
read_digital_twin('i.agency')
```
Результат:
```json
{
  "development_regularity": "5 дней/неделю",
  "execution_stability": "серии по 4 дня",
  "reflection_frequency": "5 раз/неделю",
  "strategy_regularity": "80%",
  "systemic_levels_of_intent": "Уровень команды"
}
```

#### write_digital_twin(path, data)

Записывает значение показателя в цифровой двойник.

**Параметры:**
- `path` (string): путь к показателю
- `data` (any): новое значение (должно соответствовать формату из мета-модели)

**Примеры:**

```
write_digital_twin('i.role_set', ['писатель', 'читатель', 'слушатель', 'наставник'])
```
Результат:
```json
{"status": "ok", "path": "i.role_set", "previous": ["писатель", "читатель", "слушатель"]}
```

```
write_digital_twin('i.daily_task_time', '09:00')
```
Результат:
```json
{"status": "ok", "path": "i.daily_task_time", "previous": "08:00"}
```

#### get_trainee_context()

Возвращает контекст стажёра для промптов (все показатели с `for_prompts: true`).

**Результат:**
```json
{
  "objective": "Освоить системное мышление для применения в работе",
  "role_set": ["менеджер проекта", "отец"],
  "domain_focus": ["IT", "управление"],
  "qualification_level": "Систематический",
  "daily_task_time": "08:00",
  "target_state": {"energy": 4, "attention": 3, "sleep": 4},
  "highlighted_methods": ["медленное чтение", "мышление письмом"]
}
```

## 5. Интеграция с экосистемой

### 5.1. Связь с Проводником

Проводник по персональному маршруту использует мета-модель для:

1. **Персонализации контента:**
   - `i.role_set` → примеры из ролей стажёра
   - `i.domain_focus` → контекст предметной области
   - `i.qualification_level` → уровень сложности материала

2. **Планирования отправок:**
   - `i.daily_task_time` → время отправки заданий
   - `i.daily_report_time` → время запроса отчёта
   - `i.weekly_analytics_time` → время недельной аналитики

3. **Адаптации маршрута:**
   - `i.guide_track_set` → выбранные руководства
   - `i.weekly_section` → текущий раздел
   - `i.highlighted_methods` → методы в фокусе

### 5.2. Связь с моделью созидателя (A2.2.Architecture)

Мета-модель цифрового двойника — это операционализация модели созидателя:

| Концепт из модели созидателя | Блок мета-модели |
|------------------------------|------------------|
| Роли созидателя | i.role_set |
| Компетенции | i.practice_levels, i.mastered_methods |
| Уровни мастерства | i.qualification_level |
| Состояние агентности | i.agency.* |
| Жизненное мастерство | i.mastery.* |
| Ресурсность | i.resourcefulness.* |

### 5.3. Связь с квалификационной моделью

Квалификация стажёра определяется автоматически на основе показателей:

```
Ступень = f(
  i.development_regularity,
  i.execution_stability,
  i.reflection_frequency,
  i.strategy_regularity,
  i.time_invested,
  i.practice_levels,
  ...
)
```

Периоды учёта по ступеням:
- Случайный: 1 неделя
- Практикующий: 4 недели
- Систематический: 8 недель
- Дисциплинированный: 12 недель
- Проактивный: 24 недели

## 6. Расширение мета-модели

### 6.1. Принципы расширения

При добавлении новых показателей:

1. **Определить группу:** в какой блок входит показатель
2. **Выбрать тип:** семантический, структурный, категориальный, частотный, шкальный, временной
3. **Задать формат:** конкретный формат данных
4. **Создать описание:** файл в соответствующей папке
5. **Обновить реестр:** добавить в registry.json
6. **Определить использование:** для промптов, для модели стажёра, для квалификации

### 6.2. Процесс добавления показателя

```
1. Создать файл i/<group>/<indicator>.md
2. Заполнить шаблон описания
3. Добавить запись в registry.json
4. Обновить MCP-сервер (если нужна специальная логика)
5. Обновить документацию
```

### 6.3. Версионирование

Мета-модель версионируется:
- **Мажорная версия:** изменение структуры групп, удаление показателей
- **Минорная версия:** добавление новых показателей
- **Патч:** уточнение описаний, исправление ошибок

Текущая версия: **0.1.0**
