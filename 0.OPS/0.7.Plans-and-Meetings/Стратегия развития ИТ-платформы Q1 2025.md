## 0. Назначение документа

Документ описывает стратегию развития ИТ-платформы экосистемы на Q1 2025 (январь–март), включая:

## 1. Общая концепция ИТ-платформы экосистемы

### 1.1. Ключевая идея

ИТ-платформа экосистемы — это **единая среда развития созидателей**, где:

1. **Участник совершает активные действия** → платформа фиксирует их как события (`E.*`)
2. **Цифровой двойник** накапливает данные и вычисляет характеристики развития
3. **ИИ-ассистенты** используют данные ЦД для персонализации траекторий
4. **Токеномика** вознаграждает за реальный вклад в развитие себя и экосистемы
5. **Индексатор знаний** позволяет создавать новые продукты из накопленных знаний

### 1.2. Три группы данных цифрового двойника

| Группа | Тип данных | Примеры | Использование |
|--------|------------|---------|---------------|
| **События** | События (`E.*`) | Лайк, пост, изучение раздела, ДЗ | Факты активных действий |
| **Характеристики** | Показатели (`IND.2.*`) | Систематичность, внимание, ступень | Агрегаты и расчётные показатели |
| **Контент пользователя** | Рабочие продукты (`W.*`) | Посты, комментарии, ответы на ДЗ | Сохраняются в Репозиторий знаний (2.3) в виде чанков (до 10 000 знаков) |

### 2.2. Реестр хранилищ данных

| №       | Хранилище               | Обозначение             | Описание                                                                                                                              | Статус      | Приоритет Q1 |
| ------- | ----------------------- | ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------- | ----------- | ------------ |
| **2.1** | Репозиторий руководств  | `S.GuidesRepository`    | GitHub-репозиторий с методологическими руководствами и нормативами. Первоисточник всех знаний экосистемы                              | 🟢 Работает | ⭐⭐⭐          |
| **2.2** | Цифровой двойник        | `S.DigitalTwinStorage`  | Хранилище событий, показателей и характеристик развития участников. База данных SurrealDB                                             | 🔴 Идея     | ⭐⭐⭐          |
| **2.3** | Репозиторий знаний      | `S.KnowledgeRepository` | GitHub-репозиторий с контентом пользователя (посты, комментарии, ДЗ), организованным в виде чанков (файлы до 10 000 знаков). Источник данных для Индексатора знаний | 🟡 Проект   | ⭐⭐⭐          |
| **2.4** | База эмбеддингов        | `S.EmbeddingsGraph`     | Векторные представления чанков из Репозитория знаний (2.3). Создаются Индексатором знаний (1.2). Используются ИИ-системами для семантического поиска через MCP | 🟡 Проект   | ⭐⭐⭐          |

**Описание работы с хранилищами:**

**Репозиторий знаний (`S.KnowledgeRepository`):**
- Хранит контент пользователя: посты, комментарии, ответы на ДЗ, заметки
- Контент организован в виде чанков (отдельных markdown-файлов)
- Максимальный размер чанка: 10 000 знаков
- Источники данных: 
  - Автоматический импорт из Клуба (посты, комментарии)
  - Автоматический импорт из LMS (ответы на ДЗ)
  - Ручная загрузка пользователем (заметки, рефлексии)
- Пользователь может редактировать чанки

**Индексатор знаний (`S.KnowledgeIndexer`):**

Система полного цикла преобразования первичного контента в эмбеддинги:

1. **Сбор первичного контента** из источников (Клуб, LMS, Telegram, ручной ввод)
2. **Нормализация (чанкование)** — автоматическая и ручная разбивка на чанки (до 10k знаков) и сохранение в Репозиторий знаний (2.3)
3. **Индексация (эмбеддингизация)** — преобразование чанков в векторные представления и сохранение в Базу эмбеддингов (2.4)
4. **Синхронизация** — отслеживание изменений чанков и автоматическое обновление эмбеддингов

**База эмбеддингов (`S.EmbeddingsGraph`):**
- Хранит векторные представления чанков
- Оснащена MCP-сервером для доступа ИИ-систем
- ИИ-системы используют эмбеддинги для создания информационных продуктов
- Обеспечивает семантический поиск по контенту пользователя

## 3. Обновлённая программа «Личное развитие» (ЛР)

### 3.1. Концепция: от случайного к проактивному ученику

Обновлённая программа ЛР должна провести человека через ступени роли Ученика. В процессе обучения все материалы участника автоматически сохраняются в его персональный **Репозиторий знаний (2.3)**, откуда **Индексатор знаний (1.2)** создаёт эмбеддинги для **Базы данных эмбеддингов (2.4)**. Это позволяет ИИ-ассистентам персонализировать траекторию обучения и помогать участнику более качественно.

### 3.2. Результат программы ЛР

Человек выходит из программы с:

1. **Репозиторием проекта** (SRT-структура)
   - Организовано по методу SRT (3 системы × 3 роли)
   - Настроенный процесс работы с Системным аналитиком
   - Интеграция с Индексатором знаний

2. **Настроенным процессом реализации**
   - Персональный маршрут развития (`PTH.Student.PersonalRoute`)
   - Ритм работы со слотами (`SLT.*`)
   - Интеграция с ИИ-ассистентами

3. **Прокачанными качествами**
   - Агентность и калибр личности
   - Системное мировоззрение
   - Обновлённый стиль саморазвития
   - Готовность к сложному и новому

### 3.3. ИИ-ассистенты для программы ЛР

| Ассистент | Функция | Когда используется |
|-----------|---------|-------------------|
| **ДЗ-чекер** | Проверка домашних заданий | При выполнении ДЗ |
| **Ассистент ученика** | Поддержка в обучении, ответы на вопросы | В процессе изучения |
| **Стратег** | Планирование траектории, постановка целей | При планировании, рефлексии |
| **Системный аналитик** | Организация проектов по SRT-методу, структурирование знаний | При работе с проектами |

**Системный аналитик (`AGT.SystemArchitect`)** помогает:
- Организовать хранилище проекта по SRT-методу
- Структурировать знания по 9 ячейкам (3 системы × 3 роли)
- Поддерживать связность документов
- Генерировать отчёты о состоянии проекта
- Интеграция с Репозиторием знаний (2.3) для сохранения чанков проектной документации

**Возможное объединение:** Рассмотреть создание единого «Ассистента программы ЛР», который включает функции ДЗ-чекера, Ассистента ученика, Стратега и Системного аналитика в одном интерфейсе.
