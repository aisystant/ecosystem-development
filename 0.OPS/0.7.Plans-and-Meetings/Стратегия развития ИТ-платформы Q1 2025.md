# Стратегия развития ИТ-платформы Q1 2025

## 0. Назначение документа

Документ описывает стратегию развития ИТ-платформы экосистемы на Q1 2025 (январь–март), включая:
- Общую концепцию работы ИТ-платформы
- Карту ключевых ИТ-систем и их связей
- Приоритеты и фокусы на квартал
- Детальный план работ на январь 2025

## 1. Общая концепция ИТ-платформы экосистемы

### 1.1. Ключевая идея

ИТ-платформа экосистемы — это **единая среда развития созидателей**, где:

1. **Участник совершает активные действия** → платформа фиксирует их как события (`E.*`)
2. **Цифровой двойник** накапливает данные и вычисляет характеристики развития
3. **ИИ-ассистенты** используют данные ЦД для персонализации траекторий
4. **Токеномика** вознаграждает за реальный вклад в развитие себя и экосистемы
5. **Индексатор знаний** собирает и структурирует контент участников (РП-1 → РП-4)
6. **Генератор инфопродуктов** создаёт персонализированные инфопродукты на основе Индекса знаний (РП-4 → РП-7)

### 1.2. Три группы данных цифрового двойника

| Группа | Тип данных | Примеры | Использование |
|--------|------------|---------|---------------|
| **События** | События (`E.*`) | Лайк, пост, изучение раздела, ДЗ | Факты активных действий |
| **Характеристики** | Показатели (`IND.2.*`) | Систематичность, внимание, ступень | Агрегаты и расчётные показатели |
| **Контент пользователя** | Рабочие продукты (`W.*`) | Посты, комментарии, ответы на ДЗ | Сохраняются в Репозиторий знаний (2.3) как чанки (до 10 000 знаков) |

### 2.2. Реестр хранилищ данных

| № | Хранилище | Обозначение | Описание | Статус | Приоритет Q1 |
|---|---------|-------------|----------|--------|--------------|
| **2.1** | Репозиторий S2R | `S.S2RRepository` | GitHub-репозиторий с методологическими руководствами, нормативами и описаниями систем. Первоисточник всех знаний экосистемы | 🟢 Работает | ⭐⭐⭐ |
| **2.2** | Хранилище цифрового двойника | `S.DigitalTwinStorage` | Хранилище событий, показателей и характеристик развития участников. Используется для персонализации инфопродуктов | 🔴 Идея | ⭐⭐⭐ |
| **2.3** | Репозиторий знаний | `S.KnowledgeRepository` | GitHub-репозиторий с контентом пользователя (чанки до 10 000 знаков). Источник данных для Индексатора знаний. Пользователь может редактировать чанки | 🟡 Проект | ⭐⭐⭐ |
| **2.4** | Индекс знаний | `S.KnowledgeIndex` | Структурированное хранилище: векторные представления (эмбеддинги) + лексический индекс + метаиндекс. Создаётся Индексатором знаний (1.2). Используется Генератором инфопродуктов (1.6) и другими ИИ-системами | 🟡 Проект | ⭐⭐⭐ |

**Описание работы с хранилищами:**

**Репозиторий знаний (`S.KnowledgeRepository`):**
- Хранит контент пользователя: посты, комментарии, ответы на ДЗ, заметки
- Контент организован в виде чанков (отдельных markdown-файлов с YAML frontmatter)
- Максимальный размер чанка: 10 000 знаков (без секции истории изменений)
- Источники данных:
  - Автоматический импорт из Клуба (посты, комментарии)
  - Автоматический импорт из LMS (ответы на ДЗ)
  - Ручная загрузка пользователем (заметки, рефлексии)
- Пользователь может редактировать чанки

**Индексатор знаний (`S.KnowledgeIndexer`):**

Система полного цикла преобразования первичного контента в Индекс знаний (4 подсистемы):

1. **Сборщик (Collector)** — сбор первичного контента из источников (Клуб, LMS, Telegram, ручной ввод) → РП-2
2. **Нормализатор (Normalizer/Chunker)** — автоматическое и ручное чанкование, присвоение ID и метаданных → РП-3
3. **Индексатор (Embedder)** — создание эмбеддингов, сохранение в векторное хранилище → РП-4
4. **Синхронизатор (Watcher)** — отслеживание изменений чанков, автоматическое обновление индекса

**Индекс знаний (`S.KnowledgeIndex`):**
- Хранит векторные представления чанков (эмбеддинги)
- Оснащён MCP-сервером для доступа ИИ-систем
- Используется Генератором инфопродуктов для сборки пакета контекста
- Обеспечивает семантический и лексический поиск по контенту

**Генератор инфопродуктов (`S.InfoProductGenerator`):**

Система создания инфопродуктов на основе Индекса знаний (6 подсистем):

1. **Интерпретатор запроса** — превращает запрос в Карточку запроса
2. **Планировщик контекста** — решает, где искать информацию → План подбора контекста
3. **Сборщик контекста** — собирает релевантные чанки → Пакет контекста
4. **Генератор инфопродукта** — создаёт черновик на основе контекста
5. **Проверка качества** — проверяет соответствие стандартам → Отчёт о проверке
6. **Сохранение** — размещает инфопродукт в хранилищах

Типы инфопродуктов:
- **Руководства:** персональные, универсальные, адаптивные
- **Документы для хранилищ:** правила, описания систем, карты связей
- **Документы для заинтересованных лиц:** инструкции, спецификации, отчёты
- **Коммуникационные материалы:** посты, статьи, тезисы выступлений

## 3. Обновлённая программа «Личное развитие» (ЛР)

### 3.1. Концепция: от случайного к проактивному ученику

Обновлённая программа ЛР должна провести человека через ступени роли Ученика. В процессе обучения все материалы участника автоматически сохраняются в его персональный **Репозиторий знаний (2.3)**, откуда **Индексатор знаний (1.2)** создаёт **Индекс знаний (2.4)**. **Генератор инфопродуктов (1.6)** использует Индекс для создания персонализированных руководств и других инфопродуктов.

│                                                                              │

### 3.2. Результат программы ЛР

Человек выходит из программы с:

1. **Репозиторием проекта** (SRT-структура)
   - Организовано по методу SRT (3 системы × 3 роли)
   - Настроенный процесс работы с Системным аналитиком
   - Интеграция с Индексатором знаний

2. **Настроенным процессом реализации**
   - Персональный маршрут развития (`PTH.Student.PersonalRoute`)
   - Ритм работы со слотами (`SLT.*`)
   - Интеграция с ИИ-ассистентами

3. **Прокачанными качествами**
   - Агентность и калибр личности
   - Системное мировоззрение
   - Обновлённый стиль саморазвития
   - Готовность к сложному и новому

### 3.3. ИИ-ассистенты для программы ЛР

| Ассистент | Функция | Когда используется |
|-----------|---------|-------------------|
| **ДЗ-чекер** | Проверка домашних заданий | При выполнении ДЗ |
| **Ассистент ученика** | Поддержка в обучении, ответы на вопросы | В процессе изучения |
| **Стратег** | Планирование траектории, постановка целей | При планировании, рефлексии |
| **Системный аналитик** | Организация проектов по SRT-методу, структурирование знаний | При работе с проектами |
| **Генератор инфопродуктов** | Создание персонализированных руководств и адаптивных материалов | При запросе на контент |

**Системный аналитик (`AGT.SystemArchitect`)** помогает:
- Организовать хранилище проекта по SRT-методу
- Структурировать знания по 9 ячейкам (3 системы × 3 роли)
- Поддерживать связность документов
- Генерировать отчёты о состоянии проекта
- Интеграция с Репозиторием знаний (2.3) для сохранения чанков проектной документации

**Генератор инфопродуктов (`S.InfoProductGenerator`)** создаёт:
- Персональные руководства под конкретного пользователя, его цели и ограничения
- Адаптивные руководства (ядро универсальное + персональные вставки по профилю)
- Документы для хранилищ (правила, описания систем, карты)
- Коммуникационные материалы (посты, статьи, тезисы)

**Возможное объединение:** Рассмотреть создание единого «Ассистента программы ЛР», который включает функции ДЗ-чекера, Ассистента ученика, Стратега и Системного аналитика в одном интерфейсе.
