# Глоссарий

## Системная терминология (S2R)

### Ядро (Kernel)
**Определение**: Полная 3×3 матрица "Системы × Роли", представляющая одну точку зрения на систему интереса (SoI).

**Структура**: 3 системы × 3 роли = 9 папок.

**Примеры**: A.Созидатель, B.Экосистема-развития, C.ИТ-платформа-Экосистемы

### SoI (System of Interest)
**Определение**: Система интереса — центральная система, которую мы описываем в конкретном ядре.

**Примеры**:
- В ядре A: SoI = Созидатель (A2)
- В ядре B: SoI = Экосистема-развития (B2)
- В ядре C: SoI = ИТ-платформа-Экосистемы (C2)

### Надсистема (Supersystem)
**Определение**: Система, которая физически или логически содержит SoI, задаёт контекст применения.

**Обозначение**: Позиция **1** в ядре (X1.*)

**Примеры**:
- A1: Команды-и-сообщества (надсистема для созидателя)
- B1: Мир (рынок как надсистема для экосистемы)
- C1: Экосистема-развития (надсистема для ИТ-платформы)

### Система создания (Creation System)
**Определение**: Система, которая создаёт, поддерживает и развивает SoI.

**Обозначение**: Позиция **3** в ядре (X3.*)

**Примеры**:
- A3: Экосистема-развития (создаёт условия для развития созидателя)
- B3: Система-создания-Экосистемы (команда + процессы + инфраструктура)
- C3: ИТ-служба (создаёт и поддерживает ИТ-платформу)

### Роли (Roles)

#### Meaning (Смыслы)
**Позиция**: *.1.*

**Контекст**: Зачем система существует? Ценностное предложение, миссия, видение.

**Типичные документы**: Mission, Vision, Value Proposition, Business Model

#### Architecture (Архитектура)
**Позиция**: *.2.*

**Контекст**: Как система устроена? Компоненты, модель, структура.

**Типичные документы**: System Architecture, Component Diagram, Data Model, Team Structure

#### Operations (Операции)
**Позиция**: *.3.*

**Контекст**: Как система работает? Процессы, практики, ритуалы.

**Типичные документы**: Operating Procedures, User Journey, Development Process, CI/CD

### Семейства документов (Document Families)
**Определение**: Группы документов, соответствующие ячейкам матрицы 3×3.

**Обозначение**: F1-F9

**Пример**: F5 = Architecture × SoI (архитектурные документы системы интереса)

### Метауровень (F0)
**Определение**: Уровень знаний, управляющий всеми ядрами.

**В репозитории**: Папка `0.OPS/`

**Содержит**: Модель ядер, глоссарий, правила именования, отчёты ИИ

## Отношения между системами

### ∋ (Содержит)
**Читается**: "содержит", "включает в себя"

**Контекст**: Физическое или логическое содержание. Надсистема содержит SoI.

**Примеры**:
- Команды ∋ людей-созидателей
- Сообщество ∋ участников

**Обратное**: ∈ (входит в)

### ⊃ (Включает как подсистему)
**Читается**: "включает как подсистему"

**Контекст**: Составная система включает подсистемы. SoI включает подсистемы.

**Примеры**:
- Экосистема ⊃ ИТ-платформу
- Экосистема ⊃ Сообщество
- Система создания ⊃ ИТ-службу

**Обратное**: ⊂ (является подсистемой)

### → (Создаёт)
**Читается**: "создаёт", "производит", "разрабатывает"

**Контекст**: Система создания создаёт SoI.

**Примеры**:
- ИТ-служба → ИТ-платформу
- Система создания → Экосистему
- Экосистема → Созидателя (через обучение/развитие)

**Обратное**: ← (создаётся)

## Доменная терминология

### Созидатель
**Определение**: Человек, который создаёт ценность через применение системного мышления и инженерных практик.

**Роли**:
- Разработчик
- Предприниматель
- Учащийся

**SoI в ядре**: A.Созидатель/A2.Созидатель/

### Экосистема развития
**Определение**: Платформа + Сообщество + Контент, обеспечивающие развитие созидателей.

**Подсистемы**:
- Сообщество (социальная среда)
- ИТ-платформа (техническая инфраструктура)
- Контент (знания, курсы, материалы)

**SoI в ядре**: B.Экосистема-развития/B2.Экосистема-развития/

### ИТ-платформа Экосистемы
**Определение**: Техническая инфраструктура экосистемы, включающая MCP-серверы, ИИ-агентов, репозитории знаний.

**Компоненты**:
- MCP servers (fsm-mcp, digital-twin-mcp, guides-mcp)
- ИИ-агенты (Orchestrator, RouteGuide, SystemArchitect и др.)
- Репозитории данных (Knowledge Repository, Digital Twin, FSM Models)

**SoI в ядре**: C.ИТ-платформа-Экосистемы/C2.ИТ-платформа-Экосистемы/

### ИТ-служба
**Определение**: Команда разработчиков, DevOps, SRE, создающая и поддерживающая ИТ-платформу.

**Роли**:
- Разработчики (Backend, Frontend, ML)
- DevOps-инженеры
- SRE (Site Reliability Engineering)
- Архитекторы

**Система в ядре**: C.ИТ-платформа-Экосистемы/C3.ИТ-служба/

### Команды и сообщества
**Определение**: Группы созидателей, работающие над общими проектами или объединённые общими интересами.

**Типы**:
- Проектные команды
- Сообщества практики (Communities of Practice)
- Учебные когорты
- Open Source сообщества

**Надсистема в ядре**: A.Созидатель/A1.Команды-и-сообщества/

### Система создания Экосистемы
**Определение**: Полная система, создающая экосистему, включает команды, процессы и инфраструктуру.

**Включает**:
- ИТ-службу (C3)
- Контент-команду
- Команду сообщества
- Бизнес-команду

**Система в ядре**: B.Экосистема-развития/B3.Система-создания-Экосистемы/

## Технические термины

### MCP (Model Context Protocol)
**Определение**: Протокол взаимодействия между ИИ-моделями и внешними инструментами/данными.

**В архитектуре**: Tool Layer между AI Runtime и Data Layer

**MCP-серверы в платформе**:
- **fsm-mcp**: Управление состояниями (Finite State Machines)
- **digital-twin-mcp**: Цифровой двойник созидателя
- **guides-mcp**: Репозиторий знаний и руководств

### ИИ-агент
**Определение**: Специализированный компонент с AI-возможностями, выполняющий определённую роль.

**Примеры**:
- **Orchestrator**: Координация агентов
- **RouteGuide**: Навигация по пути развития
- **SystemArchitect**: Системное проектирование
- **ProgressAnalyst**: Анализ прогресса

### Репозиторий знаний (Knowledge Repository)
**Определение**: Структурированное хранилище документов, руководств, курсов.

**Формат**: S2R multi-kernel (3 ядра: A, B, C + F0 метауровень)

**Технологии**: Git, Markdown, Frontmatter

### Цифровой двойник (Digital Twin)
**Определение**: Цифровая модель созидателя, отслеживающая прогресс, компетенции, траекторию развития.

**Данные**:
- Текущие компетенции
- Пройденные модули/курсы
- Проекты и достижения
- Траектория развития

## Методологические термины

### FPF (Full Person Framework)
**Определение**: Методология целостного развития личности, интегрированная с S2R.

**Связь с S2R**: Описана в 0.OPS/0.4.FPF-Integration/

### Системное мышление
**Определение**: Подход к пониманию сложных систем через анализ взаимосвязей между компонентами.

**Применение**: Основа S2R методологии, лежит в основе всех ядер

### Роли в команде
**Определение**: Функциональные позиции участников, определённые в методологии экосистемы.

**Документ**: A.Созидатель/A1.Команды-и-сообщества/A1.2.Architecture/ (Карта ролей)

## Правила использования терминов

### ✅ Правильно:
- "Созидатель входит в команды" (∈)
- "Экосистема включает ИТ-платформу как подсистему" (⊃)
- "ИТ-служба создаёт ИТ-платформу" (→)

### ❌ Неправильно:
- "Созидатель создаёт команды" (команды — надсистема, не создаются созидателем)
- "ИТ-платформа содержит экосистему" (наоборот)
- "Мир создаёт экосистему" (мир — надсистема, контекст, не создатель)

## Префиксы обозначений сущностей

> Расширено из "Словарь сущностей экосистемы 0.1.md"

### Система префиксов

Каждая сущность в экосистеме имеет префикс, обозначающий её тип:

| Префикс | Тип сущности | Примеры |
|---------|--------------|---------|
| **R.** | Role (Роль) | `R.GrowthGuide`, `R.Mentor`, `R.Student` |
| **S.** | System (Система, подсистема) | `S.AIPlatform`, `S.LMS`, `S.Community` |
| **AGT.** | AI Agent (ИИ-агент) | `AGT.Orchestrator`, `AGT.RouteGuide` |
| **QLF.** | Qualification (Квалификация) | `QLF.L1.Student`, `QLF.L5.Expert` |
| **C.** | Component (Компонент) | `C.Program.PersonalDevelopment` |
| **E.** | Event (Событие) | `E.LMS.SectionCompleted` |
| **IND.** | Indicator (Показатель) | `IND.2.2.7` |

**Формат**: `<Префикс>.<Имя>[.<Подтип>]`

**Примеры**:
- `AGT.RouteGuide` — агент-проводник по маршруту развития
- `R.GrowthGuide` — роль наставника-проводника
- `S.AIPlatform.MCP` — подсистема MCP в ИТ-платформе

**Подробнее**: см. файл "Словарь сущностей экосистемы 0.1.md" (старая версия, требуется актуализация под S2R)
