# Карта наших систем (Our Systems Map)

## Граф системных отношений

```
                    Мир (B1)
                      │
        ┌─────────────┴─────────────┐
        │                           │
        │                  Команды-и-сообщества (A1)
        │                           │
        │                      ∋ (содержит)
        │                           │
        ↓                           ↓
Экосистема-развития (B2) ←────── Созидатель (A2)
        │                      (создаётся)
        │
        ├─ ⊃ (включает подсистемы):
        │   ├─ Сообщество ∋ созидателей
        │   └─ ИТ-платформа-Экосистемы (C2)
        │
        ↑ (создаётся)
        │
Система-создания-Экосистемы (B3)
        │
        └─ ⊃ (включает):
            ├─ ИТ-служба (C3) → создаёт → ИТ-платформа (C2)
            └─ Другие команды
```

## Легенда символов

| Символ | Значение | Пример |
|--------|----------|--------|
| `∋` | Содержит физически | Команды ∋ людей-созидателей |
| `⊃` | Включает как подсистему | Экосистема ⊃ платформу |
| `∈` | Входит в (обратное ∋) | Созидатель ∈ команды |
| `⊂` | Является подсистемой (обратное ⊃) | Платформа ⊂ экосистемы |
| `→` | Создаёт | ИТ-служба → платформу |
| `←` | Создаётся (обратное →) | Платформа ← ИТ-службы |

## Ключевые системы

### A2: Созидатель (Target System)
**Что**: Человек — разработчик, предприниматель, учащийся

**Надсистема**: Команды-и-сообщества (A1) — созидатель входит в команды как участник

**Система создания**: Экосистема-развития (A3/B2) — экосистема создаёт условия для развития созидателя

**Ядро A** смотрит на этот мир с точки зрения созидателя.

### B2: Экосистема-развития (Creation System)
**Что**: Платформа + Сообщество + Контент для развития созидателей

**Надсистема**: Мир (B1) — рынок EdTech / Creator Economy

**Подсистемы** (в B2.2.Architecture/):
- **Сообщество** — ∋ созидателей (людей), обеспечивает социальную среду
- **ИТ-платформа-Экосистемы (C2)** — техническая инфраструктура

**Система создания**: Система-создания-Экосистемы (B3) — команда + процессы + инфраструктура

**Ядро B** смотрит на этот мир с точки зрения экосистемы как целого.

### C2: ИТ-платформа-Экосистемы (Subsystem)
**Что**: Техническая инфраструктура экосистемы (MCP servers, agents, repositories)

**Надсистема**: Экосистема-развития (C1/B2) — платформа является подсистемой экосистемы

**Система создания**: ИТ-служба (C3) — команда разработчиков, DevOps, SRE

**Ядро C** даёт детальный технический взгляд на платформу.

## Отношения между ядрами

### A ↔ B: Созидатель и Экосистема
- **A2 (Созидатель)** ∈ **A1 (Команды)**
- **B2 (Экосистема)** ⊃ **Сообщество** ∋ **A2 (Созидателей)**
- **A3 (Экосистема создания)** = **B2 (Экосистема SoI)** (разные точки зрения на одно)

**Смысл**: Созидатель участвует в командах, которые существуют в сообществе экосистемы.

### B ↔ C: Экосистема и ИТ-платформа
- **B2 (Экосистема)** ⊃ **ИТ-платформа (подсистема в B2.2.Architecture/)**
- **C2 (ИТ-платформа SoI)** ⊂ **C1 (Экосистема как надсистема)**
- **B3 (Система создания)** ⊃ **C3 (ИТ-служба)**

**Смысл**: ИТ-платформа — подсистема экосистемы, детализированная в ядре C.

### A ↔ C: Созидатель и ИТ-платформа
- **A2 (Созидатель)** использует **C2 (ИТ-платформа)** через **B2 (Экосистему)**
- Нет прямой связи — только через экосистему (ядро B)

**Смысл**: Созидатель не контактирует напрямую с инфраструктурой, только через интерфейсы экосистемы.

## Цепочка ценности (Value Chain)

```
Мир (B1) требует созидателей
    ↓
Команды (A1) нуждаются в развитых созидателях
    ↓
Созидатель (A2) развивается через экосистему
    ↓
Экосистема (B2) предоставляет платформу + сообщество
    ↓
ИТ-платформа (C2) предоставляет технические возможности
    ↓
ИТ-служба (C3) поддерживает и развивает платформу
    ↓
Система создания (B3) управляет всеми ресурсами
```

## Каскад зависимостей

### Уровень 1: Мир и Созидатель
**B1.Мир** задаёт контексты → **A1.Команды** формируются → **A2.Созидатель** участвует

### Уровень 2: Экосистема
**A2.Созидатель** требует развития → **B2.Экосистема** создаётся → **B3.Система создания** разрабатывает

### Уровень 3: ИТ-платформа
**B2.Экосистема** требует инфраструктуры → **C2.ИТ-платформа** разрабатывается → **C3.ИТ-служба** создаёт и поддерживает

## Принципы отношений

### 1. Физическое содержание (∋)
**Примеры**:
- Команды ∋ людей (физически человек находится в команде)
- Сообщество ∋ участников

**Критерий**: Можно указать на конкретный объект и сказать "он здесь".

### 2. Подсистемное включение (⊃)
**Примеры**:
- Экосистема ⊃ ИТ-платформу
- Экосистема ⊃ Сообщество
- Система создания ⊃ ИТ-службу

**Критерий**: Одна система является составной частью другой, но не физически содержится.

### 3. Создание (→)
**Примеры**:
- ИТ-служба → ИТ-платформу
- Система создания → Экосистему
- Экосистема → Созидателя (обеспечивает развитие)

**Критерий**: Одна система создаёт/поддерживает/развивает другую.

## Частые ошибки

### ❌ Неправильно:
- "Мир содержит экосистему" (надсистема не содержит SoI физически)
- "ИТ-служба создаёт экосистему" (служба — часть системы создания, не вся система)
- "ИТ-платформа входит в ИТ-службу" (наоборот: служба входит в систему создания)

### ✅ Правильно:
- "Мир — надсистема для экосистемы" (контекст применения)
- "Система создания экосистемы включает ИТ-службу" (служба — часть)
- "ИТ-платформа — подсистема экосистемы, создаваемая ИТ-службой"

## Навигация по графу

### Вопрос: "Кто создаёт созидателя?"
**Ответ**: Экосистема-развития (B2/A3) через предоставление платформы + сообщества + контента

**Путь**: B3 → B2 → A2

### Вопрос: "Кто создаёт ИТ-платформу?"
**Ответ**: ИТ-служба (C3), которая входит в Систему создания экосистемы (B3)

**Путь**: B3 ⊃ C3 → C2

### Вопрос: "Где находится созидатель?"
**Ответ**: В командах-и-сообществах (A1), которые являются частью сообщества экосистемы (B2.2)

**Путь**: A1 ∋ A2, B2 ⊃ Сообщество ∋ A2
